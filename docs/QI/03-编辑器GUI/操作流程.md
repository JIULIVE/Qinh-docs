# 编辑器操作流程（分步实战）

> 所属：[编辑器 GUI](概览.md)　·　相关：[物品编辑器](物品编辑器.md) · [槽位速查](槽位速查.md)

[物品编辑器](物品编辑器.md) 等子页讲「界面是什么」，[槽位速查](槽位速查.md) 给「点哪个槽」，本页给**常见任务的完整分步操作流**——照着点就能完成。

> 全程权限 `qinhitems.editor`。每步标注了点哪个按钮、弹出什么、输入什么。

---

## 流程 1：从零创建一件武器

```
1. 输入 /qi editor
   → 打开类型浏览器

2. 点击「Weapon（武器）」类型格
   → 打开武器的物品浏览器

3. 点击「新建物品」（WRITABLE_BOOK，槽 49）
   → 背包关闭，聊天框提示输入 ID

4. 在聊天框输入物品 ID（如 my_sword）
   → 进入物品编辑器主界面

5. 点「材质」字段 → 聊天输入 diamond_sword
6. 点「显示名」字段 → 聊天输入 <gold>试炼之刃</gold>
7. 点「品质」字段 → 聊天输入 EPIC
8. 点「Lore 添加」字段 → 聊天输入 一把试炼之剑

9. 点「保存到 YAML」（EMERALD，槽 53）
   → 校验通过，提示保存成功 + 铁砧音效

10. 退出后 /qi give my_sword 领取查看
```

> 🖼️ **[图片占位]** 创建武器的分步截图拼图　·　建议 `assets/flow-create-weapon.png`

## 流程 2：给物品加属性

```
1. 在物品编辑器中，点「属性」字段
   → 打开属性编辑器

2. 点「添加属性」（LIME_DYE，槽 49）
   → 聊天输入属性键（如 attack_damage），或从 AP 目录选

3. 聊天输入数值
   · 单值：18
   · 范围：10-20（每件随机滚，见 变量）

4. 重复 2–3 添加更多属性（crit_rate 等）

5. （可选）点「基础属性」（NETHERITE_INGOT，槽 47）
   → 进入基础值编辑器，设 base_attack_damage / base_health / base_defense

6. 返回物品编辑器 → 保存（槽 53）
```

属性机制见 [属性与数值](../02-服主指南/属性与数值.md)。需装 AttributePlus 才真正上身。

## 流程 3：给物品配左键技能

```
1. 物品编辑器 → 点「动作/技能」字段
   → 打开动作编辑器

2. 顶部触发器标签选「left_click」（槽 0–5 之一）

3. 点「添加 ref」（LIME_DYE，槽 19）
   → 打开处理器选择器

4. 选一个处理器，例如 qi:title
   → 若该处理器有载荷 schema：打开载荷表单
       · 逐个字段点击 → 聊天输入
       · 点「确认并写入」（EMERALD，槽 49）
   → 若无 schema：直接聊天输入裸 payload

5. 重复 3–4 再加 qi:sound 等处理器

6. 点「冷却」（CLOCK，槽 21）：左键 +1s 调到 3s

7. 点「保存 actions」（EMERALD，槽 49）
   → 写回物品并热重载动作表，立即生效
```

> 提示：动作编辑器里处理器列表左键上移、右键删除、Shift+右键编辑载荷。槽位见 [槽位速查](槽位速查.md#动作编辑器-actionrefseditiongui)。

## 流程 4：把技能换成 QinhSkills 技能

```
1. 动作编辑器 → 点「添加技能」（NETHER_STAR，槽 20）
   → 打开技能绑定界面

2. 选触发器标签（left_click 等）

3. 从技能列表点一个 QinhSkills 技能
   （或点「手动输入技能 ID」ANVIL 槽 37 聊天输入）

4. （可选）点「JSON 表单」（槽 38）设 skill + level

5. 点「保存到物品」（EMERALD，槽 49）
```

等价于在 `left_click` 的 refs 加 `qinhskills:cast`。需装 QinhSkills，见 [集成](../04-开发者/集成.md)。

## 流程 5：创建并编排一套套装

```
1. 先把套装件物品都建好（流程 1），命名带统一前缀（如 warrior_helmet, warrior_sword）

2. 打开任一套装件的物品编辑器 → 点「套装」
   → 套装中心

3. 点「创建新套装」（NETHERITE_CHESTPLATE，槽 20）
   → 聊天输入 套装ID 和 显示名

4. 点「编辑该套装」（WRITABLE_BOOK，槽 20）→ 套装编辑界面

5. 点「成员物品」（CHEST，槽 23）→ 添加 warrior_helmet, warrior_sword …
   （也可只加前缀 warrior 让 warrior_xxx 全部匹配）

6. 点「档位 Bonuses」（BEACON，槽 25）→ 新建档位
   · 「2 件」→ 属性 物理伤害=5
   · 「4 件」→ 属性 物理伤害=10、药水效果 SPEED=1、技能 right_click→爆发

7. 完成（套装自动保存到 sets/套装ID.yml）
```

套装机制见 [套装](../02-服主指南/套装.md)，按钮槽位见 [槽位速查](槽位速查.md#套装-sethubgui)。

## 流程 6：给装备开宝石孔

```
1. 物品编辑器 → 点「宝石孔」（或编辑器 Hub 的 AMETHYST_SHARD）
   → 宝石孔编辑器

2. 点「添加孔位」（AMETHYST_SHARD，槽 40）
   → 聊天输入孔类型 ID（如 绿色 / qi_weapon，见 gem_socket_types.yml）

3. 重复添加更多孔；右键添加按钮可删最后一个

4. （可选）点「Legendinlay 元数据」（槽 6）→ 输入 set,slot,piece

5. 点「保存」（EMERALD，槽 53）
```

孔类型定义见 [宝石孔](../02-服主指南/宝石孔.md)。

## 流程 7：设为灵魂绑定

绑定一般靠物品 `bind_on_acquire` + 拾取，或命令。GUI 侧：

```
1. 物品编辑器 → 点「限制」→ 限制编辑器
2. 点「获得即绑定」（CHAIN/TRIPWIRE_HOOK，槽 48）切换 bind_on_acquire
3. 保存
```

或手持物品执行 `/qi soulbind [等级]` 直接绑定到自己。见 [灵魂绑定](../02-服主指南/灵魂绑定.md)。

---

## 流程 8：编辑已有物品

```
1. /qi editor → 选类型 → 物品浏览器
2. 右键目标物品 → 直接进物品编辑器
   （左键是「获取到背包」，不是编辑）
3. 改字段 → 保存
```

或直接 `/qi editor <物品ID>` 一步到位。

---

## 通用注意

- **关背包 = 丢弃未保存改动**（工作副本不自动落盘）。
- **聊天输入中途**想取消，输入 `cancel`。
- 保存时若有**校验错误**会提示并拒绝（见 [校验报错速查](../05-参考/校验报错速查.md)）。
- GUI 配的就是 YAML，保存后可在 `items/*.yml` 看到；手写后 `/qi reload` 也能在 GUI 打开。
