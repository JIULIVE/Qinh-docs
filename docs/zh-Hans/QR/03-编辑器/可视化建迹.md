本页带你**从进编辑器到存盘**走一遍：用[可视化编辑器](./editor-overview.md)给一座遗迹标出刷怪点、解锁宝箱、钥石核心。机关另成一页（[机关编辑](./mechanism-editing.md)）。

> 前提：你已经用 `/qr pos1 / pos2 / save <模板id>` 把结构存成模板了（见 [选区与保存](../02-server-guide/selection-save.md)）。本页所有命令权限为 `qinhruins.admin`。

## 一、进入编辑：`/qr editor start <模板id>`

```
/qr editor start ancient_tower
```

执行后：

1. **放参考遗迹**：QR 在你附近 128 格内找该模板的现有遗迹；找不到就**自动在你脚下生成一座**该模板的参考遗迹，让你对着它标点。聊天会提示「已自动放置参考遗迹……编辑完 `/qr remove <id>` 可清掉」。
2. **收背包、发工具**：你的物品栏被临时收起，换成一排编辑工具（退出时原样还你）。
3. **落在主菜单**：右键工具进各子编辑。

::: tip 提示
所有标记的相对坐标，都是相对**这座参考遗迹的锚点位置**算的——锚点 = 结构最小角 = (0,0,0)。所以请**对着参考遗迹本体**标点，别跑到别的遗迹上去标。
:::

::: warning 注意
如果聊天提示「自动放置参考遗迹失败」，多半是你脚下的地形不满足该模板的生成条件（如水下 / 空中限制）。换个开阔地再 `start`，或先手动 `/qr spawn` 一座该模板的遗迹、走近它再 `start`（会复用附近那座）。
:::

## 二、三种放置模式（`mode`）

「**当前模式**」决定**标记笔左键方块时放出哪种标记**。切模式有两种等价方式：

- **菜单**：在主菜单右键对应工具（僵尸头=刷怪点 / 箱子=宝箱 / 磁石=核心）。
- **命令**：`/qr editor mode <spawn|chest|core>`。

| mode | 标的是什么 | 写进蓝图的字段 |
|---|---|---|
| `spawn` | 刷怪点：在这格刷什么怪、几只、第几阶段 | `spawn-points` |
| `chest` | 解锁式奖励箱：用哪张战利品表、通关到第几阶段才开 | `loot-chests` |
| `core` | 遗迹核心：钥石右键此点激活[秘境](../02-server-guide/realms-keystones.md) | `cores` |

> 机关（`mechanisms`）不在这三种里——它走机关面板 / `/qr editor mech`，见 [机关编辑](./mechanism-editing.md)。

进入子编辑屏后，**槽 0 是标记笔（烈焰棒）**，槽 8 是「↩返回主菜单」，中间是该模式的参数工具。

## 三、`set`：设默认值

放标记前，先把「这一批要标的怪 / 数量 / 阶段 / 奖励表 / 解锁阶段」设成默认值；之后每次左键方块放的标记都套用这套默认值。改默认值有两种等价方式：

- **菜单**：右键 / 左键对应参数工具调整。
- **命令**：`/qr editor set <字段> <值>`。

| 字段（命令） | 含义 | 对应蓝图字段 | 菜单工具与操作 |
|---|---|---|---|
| `mob` | 默认怪物 ID | `spawn-points[].mob` | 「怪物」工具(刷怪点屏) 右键 → 聊天输入 |
| `count` | 默认刷怪数量 | `spawn-points[].count` | 「数量」工具 左键+1 / 右键-1 / 潜行+左键+10 / 潜行+右键-10 |
| `stage` | 默认刷怪阶段 | `spawn-points[].stage` | 「刷怪阶段」工具 左键+1 / 右键-1 |
| `loot` | 默认奖励表名 | `loot-chests[].loot-table` | 「奖励表」工具(宝箱屏) 左键下一个 / 右键上一个 / 潜行+左键聊天输入 |
| `unlock` | 默认解锁阶段 | `loot-chests[].unlock-stage` | 「解锁阶段」工具 左键+1 / 右键-1 |

示例（命令版）：

```
/qr editor set mob mm-FireSkeleton
/qr editor set count 3
/qr editor set stage 2
```

- `mob` 写法和蓝图一致：原版实体大写名（`ZOMBIE`），或 MythicMobs 怪 `mm-<MM名>`（见 [蓝图与目标 §三](../02-server-guide/blueprint-objectives.md#三spawn-points刷怪点)）。
- `count / stage / unlock` 必须是正整数，非法值会提示「请输入有效数字」。

::: tip 提示
怪物等级（`level`）在编辑器里固定写 `1`。要按秘境层数缩放的怪，把它做成 MythicMobs 怪——`level` 之后可在 `blueprint.yml` 手改，或交给秘境提 level。
:::

## 四、放标记：左键方块

设好默认值后，**手持标记笔（槽 0），左键你想放标记的那格方块**：

- 编辑器读你点的方块的世界坐标，**减去参考遗迹锚点坐标**，自动换算成相对结构原点的 `x / y / z` 写进会话。你**完全不用手算坐标**。
- 每放一个，聊天即时回显这个标记的 id、内容与相对坐标，例如：
  - 刷怪点：`刷怪点 sp1 ZOMBIE x3 阶段1 (3,2,5)`
  - 宝箱：`宝箱 c1 表=vault 解锁阶段1 (3,2,4)`
  - 核心：`遗迹核心 core1 (6,1,6) 右键它+钥石可激活秘境`
- **标记 id 自动生成**：刷怪点 `sp1 / sp2 …`、宝箱 `c1 / c2 …`、核心 `core1 / core2 …`（核心只有坐标无 id，靠序号定位）。

::: warning 注意
必须在**参考遗迹所在世界**标点；点到别的世界会提示「请在参考遗迹所在世界标记」。若参考锚点丢失（被回收 / 被删），会提示重新 `start`。
:::

### 各模式标点要点

- **刷怪点（spawn）**：点你想让怪**出生的那格**。怪会从该格中心、`y+1` 处生成（落在该格上方，避免卡地里）。先 `set mob/count/stage` 再批量点，同阶段同怪一口气标完很顺手。
- **宝箱（chest）**：点你**放奖励箱的那格**。先 `set loot`（选战利品表）和 `set unlock`（解锁阶段）。编辑器写出的宝箱默认 `per-player-once: true`、`growth-scaled: true`（每人独立一份、随成长缩放）；想改成全服共享真宝箱，存盘后手改 YAML（见 [蓝图与目标 §四](../02-server-guide/blueprint-objectives.md#四loot-chests奖励箱)）。
- **核心（core）**：点你**当作钥石交互点的醒目方块**（祭坛 / 信标 / 传送门框等）。可标多个，右键任意一个都算激活。

::: tip 提示
阶段（stage）只是个编号，要让「打完阶段 1 才推进到阶段 2」真正生效，还得配 `objectives`（分阶段击杀目标）。编辑器目前不直接编辑 objectives，存盘后在 `blueprint.yml` 补 `objectives.stages` 即可（见 [蓝图与目标 §六](../02-server-guide/blueprint-objectives.md#六objectives分阶段目标)）；已有的 objectives 在编辑期间会被原样保留、随 save 一起写回。
:::

## 五、查看与删除（`list` / `remove`）

### 列出已标的一切

```
/qr editor list
```

按「刷怪点 / 宝箱 / 机关 / 核心」分组列出当前会话里的全部标记，每条带 id 和坐标。

### 删除单个标记

```
/qr editor remove sp1
/qr editor remove c2
/qr editor remove core1
```

`remove <id>` 按 id 删（刷怪点 `spN` / 宝箱 `cN` / 核心 `coreN` / 机关用机关 id）。删成功提示「已移除标记」，找不到提示「未找到标记」。

### 用 GUI 删（更直观）

在主菜单右键「**标记总览 / 删除**」工具（书），打开 54 格列表面板：**点任一条目即删**，右下角屏障「返回编辑」关闭。

## 六、存盘（`save`）

标完一切，**写回蓝图**：

- **菜单**：主菜单右键「保存并退出」工具（下界之星）。
- **命令**：`/qr editor save`。

存盘做三件事：

1. 把会话内容**整份序列化写回** `templates/<模板id>/blueprint.yml`（`spawn-points` / `loot-chests` / `cores`，以及保留的 `objectives` / `mechanisms` / `variants` 等）。
2. **热重载**蓝图、宝箱、机关、核心注册表——立即对世界里的该模板遗迹生效。
3. **把你的真实背包还给你**，结束编辑会话。

存盘后聊天回报：`已保存蓝图 <id> (刷怪点N/宝箱N/机关N)`。

::: tip 提示
若存盘时还有没收尾的机关构建（mechBuilder），save 会**自动尝试收尾**并并入蓝图（见 [机关编辑](./mechanism-editing.md)）。
:::

### 放弃编辑

```
/qr editor cancel
```

或主菜单右键「放弃退出」工具（屏障）。**不写盘**、还你背包、结束会话。本次所有改动丢弃。

::: warning 注意
退出游戏 / 掉线会**自动 cancel**（不保存）。重要改动记得先 `save`。
:::

## 七、一条龙示例：标一座两阶段遗迹

```
# 1. 进编辑（脚下会冒出一座参考遗迹）
/qr editor start ancient_tower

# 2. 标第一阶段的小怪
/qr editor mode spawn
/qr editor set mob ZOMBIE
/qr editor set count 3
/qr editor set stage 1
（手持标记笔，左键 3 处你想刷僵尸的方块 → sp1 sp2 sp3）

# 3. 标第二阶段的 Boss
/qr editor set mob mm-FireSkeleton
/qr editor set count 1
/qr editor set stage 2
（左键 Boss 出生格 → sp4）

# 4. 标关底解锁宝箱
/qr editor mode chest
/qr editor set loot vault
/qr editor set unlock 1
（左键放宝箱的格 → c1）

# 5. 标钥石核心
/qr editor mode core
（左键祭坛方块 → core1）

# 6. 核对
/qr editor list

# 7. 存盘
/qr editor save

# 8.（可选）清掉脚下的参考遗迹
/qr remove <start 时提示的锚点id>
```

存盘后想要真正的「打怪 → 推进阶段 → 通关开箱」，到 `blueprint.yml` 补上 `objectives.stages`（见 [蓝图与目标 §六](../02-server-guide/blueprint-objectives.md#六objectives分阶段目标)），再 `/qr reload`。
