> 所属：[可视化编辑器](./editor-overview.md)　·　子页：[可视化建迹](./visual-building.md) · [机关编辑](./mechanism-editing.md)
>
> 相关：[蓝图与目标](../02-server-guide/blueprint-objectives.md) · [选区与保存](../02-server-guide/selection-save.md) · [机关系统](../02-server-guide/mechanisms.md) · [命令大全](../05-reference/commands.md)

---

QR 提供一套**游戏内可视化编辑器**：你站在世界里，对着方块点一点，就能给一座遗迹标刷怪点、宝箱、核心、机关，一键存成 `blueprint.yml`，**全程不用手写一行 YAML**。

它解决的就是「[蓝图与目标](../02-server-guide/blueprint-objectives.md)」里那件最容易出错的事——**算相对坐标**。手写蓝图时，每个点位的 `x / y / z` 都得相对结构原点（结构最小角 = 0,0,0）量出来，多一格少一格都会让怪刷到墙里、宝箱埋进地里。编辑器让你**直接左键世界里的真实方块**，它替你把世界坐标换算成相对结构原点的坐标写进蓝图。

> 🖼️ **[图片占位]** 服主手持「标记笔」对着遗迹方块点出刷怪点 / 宝箱的现场截图　·　建议 `assets/editor-overview.png`

## 一、它和手写 YAML 是什么关系

**完全等价、可混用。** 编辑器读写的就是 `templates/<模板id>/blueprint.yml` 这同一份文件：

- `/qr editor start` 进编辑时，会把现有 `blueprint.yml` 的内容**全部读进编辑会话**（已有的刷怪点 / 宝箱 / 核心 / 机关都在），你接着改。
- `/qr editor save` 时把会话内容**整份写回** `blueprint.yml`（由 `BlueprintWriter` 序列化），然后热重载相关注册表。
- 你也可以手写一半 YAML、再用编辑器补点位；或编辑器标完点、再手动微调 YAML。两种方式针对的是同一份文件，互不冲突。

蓝图的每个字段含义见 [蓝图与目标](../02-server-guide/blueprint-objectives.md)，机关字段见 [机关系统](../02-server-guide/mechanisms.md)——本组文档只讲「怎么在游戏里点出来」。

::: warning 注意
`save` 会用**当前编辑会话的全部内容**覆盖写 `blueprint.yml`。如果你在编辑期间手动改过磁盘上的同一个文件，编辑器的保存会覆盖掉你的手改。要么纯手写、要么纯用编辑器，别在同一座遗迹的同一次编辑里两头改。
:::

## 二、整体工作流

一座遗迹从结构文件到能玩，分两段：先**圈结构**存成模板，再**进编辑器标玩法**。

```text
① 框选结构（见 选区与保存）
   /qr pos1 + /qr pos2  → 框住遗迹方块本体
   /qr save <模板id>    → 存出结构文件 + 生成模板骨架
        │
        ▼
② 可视化标玩法（本组文档）
   /qr editor start <模板id>
        │  自动在你脚下放一座「参考遗迹」供对照（附近 128 格内已有则复用）
        │  你的物品栏被临时收起，换成一排「编辑工具」
        ▼
   右键工具切到「刷怪点 / 宝箱 / 核心 / 机关」子编辑
        │
        ▼
   左键世界里的方块 = 在那一格放一个标记（坐标自动换算成相对结构原点）
        │
        ▼
   /qr editor save（或右键「保存」工具）
        → 写回 templates/<模板id>/blueprint.yml + 热重载，物品栏还给你
```

> 第 ① 段（pos1 / pos2 / save / 结构原点）属于「造遗迹」范畴，详见 [选区与保存](../02-server-guide/selection-save.md)；本组文档从 `/qr editor start` 之后开始讲。

## 三、进编辑器时发生了什么

执行 `/qr editor start <模板id>` 后：

1. **找参考遗迹**：在你附近 128 格内找该模板的现有锚点；找不到就**自动在你脚下生成一座**该模板的参考遗迹，供你对着标点（编辑完可 `/qr remove <id>` 清掉它）。所有标记的相对坐标都是相对**这座参考遗迹的锚点位置**算的。
2. **收起背包、发放编辑工具**：你的物品栏内容被临时存起来，换成一排编辑工具（见下文「编辑工具」）。退出编辑（保存或取消）时**原样还给你**。
3. **进入主菜单**：默认在「主菜单」屏，物品栏就是菜单，右键工具进各子编辑。

> 编辑期间禁止丢弃物品、禁止换副手，退出游戏会自动结束编辑（不保存）。这些都是为了保护你的真实背包不和编辑工具混在一起。

## 四、两种操作方式：物品栏菜单 vs 命令

编辑器有**两套等价入口**，做的是同一件事，随你混用：

| 方式 | 怎么用 | 适合 |
|---|---|---|
| **物品栏菜单（推荐）** | 右键工具切屏 / 调参数，左键方块放标记，开 GUI 面板删标记、配机关 | 边走边标、所见即所得 |
| **命令** | `/qr editor mode|set|list|remove|mech|save|cancel` | 精确设值、脚本化、远程调参 |

两者共享同一个编辑会话——你用菜单切到「刷怪点」屏，等同于 `/qr editor mode spawn`；命令 `/qr editor set count 3` 改的默认数量，菜单里的数量工具也会同步显示。

## 五、命令树速览

全部命令权限为 `qinhruins.admin`，且**必须在游戏内、且已 `start` 进编辑**才有意义（`list / remove / mech / set / mode / save / cancel` 都要求你正在编辑中）。

| 命令 | 作用 | 详见 |
|---|---|---|
| `/qr editor start <模板id>` | 进入编辑：放参考遗迹、收背包、发工具 | [可视化建迹 §一](./visual-building.md#一进入编辑qr-editor-start-模板id) |
| `/qr editor mode <spawn\|chest\|core>` | 切换放置模式（决定左键放哪种标记） | [可视化建迹 §二](./visual-building.md#二三种放置模式mode) |
| `/qr editor set <字段> <值>` | 设当前默认值，字段 = `mob` / `count` / `stage` / `loot` / `unlock` | [可视化建迹 §三](./visual-building.md#三set设默认值) |
| `/qr editor mech <子命令...>` | 机关构建（new / trigger / action / flag / done …） | [机关编辑](./mechanism-editing.md) |
| `/qr editor list` | 列出当前会话已标的刷怪点 / 宝箱 / 机关 / 核心 | [可视化建迹 §五](./visual-building.md#五查看与删除list--remove) |
| `/qr editor remove <id>` | 删除指定标记（如 `sp1` / `c1` / `core1` / 机关id） | [可视化建迹 §五](./visual-building.md#五查看与删除list--remove) |
| `/qr editor save` | 写回 `blueprint.yml` + 热重载 + 还背包 | [可视化建迹 §六](./visual-building.md#六存盘save) |
| `/qr editor cancel` | 放弃编辑（不写盘）+ 还背包 | [可视化建迹 §六](./visual-building.md#六存盘save) |

> 注：菜单里「机关」子编辑可达，但 `mode` 命令的可选值只暴露 `spawn / chest / core` 三种——机关走 `/qr editor mech` 子命令或菜单的机关面板进，详见 [机关编辑](./mechanism-editing.md)。

## 六、编辑工具（物品栏菜单）

`start` 后你的物品栏被换成一排带特殊标记的「编辑工具」（它们由 CoreLib 打 NBT 标识，不是真物品，退出即收回）。

### 主菜单屏

| 工具 | 材质 | 右键作用 |
|---|---|---|
| 刷怪点编辑 | 僵尸头 | 进「刷怪点」子编辑 |
| 宝箱编辑 | 箱子 | 进「宝箱」子编辑 |
| 遗迹核心编辑 | 磁石 | 进「核心」子编辑 |
| 机关编辑 | 红石 | 进「机关」子编辑 |
| 标记总览 / 删除 | 书 | 打开 54 格列表 GUI，点条目即删 |
| 保存并退出 | 下界之星 | 写盘 + 还背包（= `/qr editor save`） |
| 放弃退出 | 屏障 | 不保存退出（= `/qr editor cancel`） |

### 子编辑屏（各自一排工具）

进任一子编辑后，槽 0 永远是「**标记笔**」（烈焰棒）——**左键方块**就在那一格放当前模式的标记；其余槽是该模式的参数工具（怪物 / 数量 / 阶段 / 奖励表 / 解锁阶段 / 机关面板），槽 8 是「↩返回主菜单」。每种模式标什么、参数怎么调，见 [可视化建迹](./visual-building.md)。

### 两个 GUI 面板

- **标记总览（54 格箱子界面）**：列出所有已标的刷怪点 / 宝箱 / 核心 / 机关，**点任一条目即删**，右下角「返回编辑」关闭。
- **机关面板（27 格箱子界面）**：可视化选触发器、加动作、完成机关，详见 [机关编辑](./mechanism-editing.md)。

> 聊天输入：怪物 ID、奖励表名、机关材质 / 文字 / 命令等**自由文本**，点对应工具后会**关背包、提示在聊天框输入**，输入 `取消`（或 `cancel`）放弃。由 `ChatInputService` 驱动。
