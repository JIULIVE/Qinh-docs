# 权限节点

> 所属：[参考手册](./commands.md)　·　相关：[命令大全](./commands.md)　·　返回：[首页](../README.md)

QR 只有**两个**权限节点，结构极简：一个面向所有玩家的基础权限、一个面向管理员的总权限。核对自 `plugin.yml`。

---

## 权限节点总表

| 节点 | 默认 | 说明 |
|---|---|---|
| `qinhruins.use` | `true`（所有人） | 使用 QR 基础命令（list / near / activate / codex / reroll / reward / party / guide cancel） |
| `qinhruins.admin` | `op`（仅管理员） | QR 全部管理权限。**自动包含 `qinhruins.use`**（`children: qinhruins.use: true`） |

> `qinhruins.admin` 的 `children` 配置意味着：给了 admin 就自动拥有 use，**无需单独再授 use**。

---

## 各命令所需权限

### 无需权限

| 命令 | 备注 |
|---|---|
| `/qr`、`/qr help` | 帮助菜单，源码未挂权限，任何人可看 |

### `qinhruins.use`（玩家）

`version`、`list`、`near`、`activate`、`codex`、`reroll`、`reward`、`party create`、`party invite`、`party accept`、`party leave`、`party info`、`guide cancel`。

### `qinhruins.admin`（管理员）

`reload`、`pos1`、`pos2`、`save`、`spawn`、`gentest`、`why`、`scatter`、`stage *`、`import`、`tp`、`remove`、`markers`、`guide clear`、`keystone give`、`genstruct`、`editor *`、`info`、`profile`。

> `stage *` 与 `editor *` 的子命令在源码中继承自各自父节点（`stage` / `editor`）声明的 `qinhruins.admin`，因此整组都需要管理员权限。

完整逐条对照见 [命令大全](./commands.md)。

---

## 权限组配置建议

### 普通玩家组（default）

无需任何配置——`qinhruins.use` 默认 `true`，所有玩家开箱即用基础探索 / 秘境 / 组队 / 图鉴功能。

若想**收紧**（例如关闭普通玩家自助激活秘境），用权限插件显式取消：

```yaml
# LuckPerms 示例：默认组撤销 use
permissions:
  - qinhruins.use: false
```

### 建迹 / 运营组（builder）

只想让某些 GM 能造遗迹、调生成、发钥石，但不想给整套 OP：

```yaml
# 直接授予 admin 总权限（已自动含 use）
permissions:
  - qinhruins.admin: true
```

QR 没有更细粒度的子权限，`qinhruins.admin` 是「全有或全无」。如需把「发钥石」「重载」分人管理，请在权限插件层面用命令黑白名单（如 LuckPerms 的 `command` 上下文）控制，QR 自身不拆分。

### 保护绕过

遗迹保护（[刷怪与通关](../02-server-guide/spawning-clearing.md)）对 `qinhruins.admin` 持有者**自动放行**：管理员可在探索 / 激活中的遗迹内破坏放置方块，便于现场调试。给建迹组授 admin 即同时获得绕过能力。
