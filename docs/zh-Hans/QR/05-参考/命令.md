# 命令大全

> 所属：[参考手册](./commands.md)　·　相关：[权限节点](./permissions.md) · [诊断排错](./troubleshooting.md)　·　返回：[首页](../README.md)

根命令：`/qr`，别名 `/qruins`。下表为完整命令树，逐条核对自源码 `QinhRuinsCommands.kt`。

命令分两组：**玩家命令**（权限 `qinhruins.use`，默认所有人可用）与**管理员命令**（权限 `qinhruins.admin`，默认仅 OP，且自动含 `use`）。权限组配置见 [权限节点](./permissions.md)。

> 约定：`<必填>`、`[可选]`。绝大多数命令需玩家在游戏内执行（涉及位置 / 背包）；纯查询类命令控制台亦可。

---

## 玩家命令（`qinhruins.use`）

| 命令 | 权限 | 作用 |
|---|---|---|
| `/qr` | — | 打开帮助菜单（无需权限） |
| `/qr help` | — | 同上，帮助菜单 |
| `/qr version` | `qinhruins.use` | 显示 QR 版本号 |
| `/qr list` | `qinhruins.use` | 列出所有模板（含生成模式）与当前所有锚点（坐标 + 状态） |
| `/qr near [radius]` | `qinhruins.use` | 列出附近遗迹锚点（按距离），`radius` 单位格，缺省 200 |
| `/qr activate` | `qinhruins.use` | 激活脚下 / 附近遗迹为秘境（搜索半径 `realm.activate-radius`，默认 48）。需手持钥石或满足激活条件 |
| `/qr codex` | `qinhruins.use` | 打开图鉴 GUI（已发现遗迹的攻略卡收集册） |
| `/qr reroll` | `qinhruins.use` | 花货币重铸手中钥石的词缀（需 `realm.reroll.enabled: true`） |
| `/qr reward` | `qinhruins.use` | 打开最近秘境的净化奖励老虎机（领取个人独占战利品） |

### 组队（`qinhruins.use`）

| 命令 | 作用 |
|---|---|
| `/qr party create` | 创建内置队伍（成为队长） |
| `/qr party invite <player>` | 邀请在线玩家加入队伍 |
| `/qr party accept` | 接受最近收到的队伍邀请 |
| `/qr party leave` | 退出当前队伍 |
| `/qr party info` | 查看队伍成员列表 |

> 组队玩法（共享进度 / 集结半径）见 [组队与会话](../02-server-guide/party-sessions.md)。若装有 QinhClass / MMOCore，队伍可由它们接管。

### 向导（玩家侧）

| 命令 | 权限 | 作用 |
|---|---|---|
| `/qr guide cancel` | `qinhruins.use` | 取消正在进行的指引并返还指引物。可在客户端把任意键（如 K）绑到此命令 |

> 指引物由物品源给予（`qinhruins:guide_<模板id>`），QR **不内置 give 命令**。详见 [向导与图鉴](../02-server-guide/guide-codex.md)。

## 管理员命令（`qinhruins.admin`）

### 重载

| 命令 | 作用 |
|---|---|
| `/qr reload` | 重载 config.yml、语言、放置档案、模板、蓝图、调色板、战利品表、词缀、层数 / 钥石 / 重铸配置，并重建锚点 / 宝箱 / 核心 / 机关注册与各调度任务 |

### 选区与保存（把建筑变遗迹）

| 命令 | 参数 | 作用 |
|---|---|---|
| `/qr pos1` | — | 把准星瞄准的方块（100 格内）设为选区第一角 |
| `/qr pos2` | — | 设为选区第二角 |
| `/qr save <id>` | `id`=新模板 id | 把 pos1~pos2 选区存为模板 `templates/<id>/`（含结构文件与蓝图骨架） |
| `/qr import <file> <id>` | `file`=`schematics/` 下 `.schem` 文件名；`id`=新模板 id | 把 WorldEdit `.schem` 导入并转为 `.nbt` 模板 |

详见 [选区与保存](../02-server-guide/selection-save.md) · [结构文件](../02-server-guide/structure-files.md)。

### 生成与投放

| 命令 | 参数 | 作用 |
|---|---|---|
| `/qr spawn <模板>` | `模板`=模板 id | 在脚下立刻手动投放一座该模板的遗迹（生成一个锚点） |
| `/qr gentest` | — | 在身边（约 25 格）触发一次生成总控调试生成，并回报生成摘要 |
| `/qr why <模板>` | `模板`=模板 id | 诊断「为什么这里生成 / 不生成该模板」：逐条列出维度 / 群系 / 平整度 / 间距等放置条件的通过情况 |
| `/qr scatter <radius> <count>` | `radius` 100~20000；`count` 1~200 | 在半径内预生成（撒播）最多 count 座遗迹，用于初始铺图 |

详见 [自然生成与预加载](../02-server-guide/generation-preloading.md)。

### 预览暂存（stage）

放置前用幽灵方块在原地预览结构本体（仅自己可见，不改世界）。详见 [选区与保存](../02-server-guide/selection-save.md)。

| 命令 | 参数 | 作用 |
|---|---|---|
| `/qr stage start <模板>` | `模板`=模板 id | 开始在脚下预览该模板 |
| `/qr stage here` | — | 把预览移到当前位置 |
| `/qr stage move <dx> <dy> <dz>` | 三个整数偏移 | 按偏移微调预览位置 |
| `/qr stage rotate` | — | 旋转预览 90° |
| `/qr stage confirm` | — | 确认放置（落地为真实结构） |
| `/qr stage cancel` | — | 取消预览 |

> `stage` 各子命令在源码里继承自 `stage` 父节点的 `qinhruins.admin` 权限。

### 锚点管理

| 命令 | 参数 | 作用 |
|---|---|---|
| `/qr tp <锚点>` | `锚点`=锚点 id | 传送到指定锚点 |
| `/qr remove <锚点>` | `锚点`=锚点 id | 回收锚点（若有快照则还原地形，腾出密度名额） |
| `/qr markers <锚点>` | `锚点`=锚点 id | 扫描该锚点结构内的标记块（刷怪点 / 宝箱 / 核心 / 命令），写回蓝图并重新捕获结构 |
| `/qr info [锚点]` | 缺省=全局状态；带 `锚点` 看单座详情 | 无参：模板 / 锚点 / 会话 / 跟踪数概览。带锚点：模板、坐标、状态、会话忙闲、蓝图刷怪点 / 宝箱 / 阶段数、目标进度 |

标记块说明见 [结构文件](../02-server-guide/structure-files.md)。

### 向导（管理侧）

| 命令 | 参数 | 作用 |
|---|---|---|
| `/qr guide clear <player>` | `player`=玩家名 | 强制清除某玩家的指引状态（不返还物品） |

### 秘境 / 词缀 / 程序化

| 命令 | 参数 | 作用 |
|---|---|---|
| `/qr keystone give <tier> [player]` | `tier` 1~`realm.tiers.max`；`player` 缺省=自己 | 发放指定层数的钥石 |
| `/qr genstruct <palette> [dry\|seed]` | `palette`=调色板 id；`dry`=只规划不放置；或填整数当随机种子 | 用调色板程序化拼接生成大遗迹（`dry` 仅打印拼接计划） |

详见 [秘境与钥石](../02-server-guide/realms-keystones.md) · [词缀系统](../02-server-guide/affixes.md) · [程序化生成](../02-server-guide/procedural-generation.md)。

### 可视化编辑器（editor）

站在世界里框选、标点、配机关，一键存模板，全程不碰 YAML。详见 [可视化编辑器](../03-editor/editor-overview.md)。

| 命令 | 参数 | 作用 |
|---|---|---|
| `/qr editor start <模板>` | `模板`=模板 id | 进入编辑会话，绑定到该模板 |
| `/qr editor mode <spawn\|chest\|core>` | 三种模式之一 | 切换标点模式：刷怪点 / 宝箱 / 核心 |
| `/qr editor set <field> <value>` | `field`=`mob`/`count`/`stage`/`loot`/`unlock`；`value` | 设置当前标点的字段 |
| `/qr editor mech <args>` | 机关参数（贪婪字符串） | 编辑机关，参数见 [机关编辑](../03-editor/mechanism-editing.md) |
| `/qr editor list` | — | 列出当前会话已标的点 |
| `/qr editor remove <id>` | `id`=标点 id | 移除一个标点 |
| `/qr editor save` | — | 保存编辑会话到模板 |
| `/qr editor cancel` | — | 取消编辑会话 |

> `editor` 各子命令继承自 `editor` 父节点的 `qinhruins.admin` 权限。

### 诊断 / 性能

| 命令 | 参数 | 作用 |
|---|---|---|
| `/qr profile [reset]` | 缺省=查看；`reset`=清零 | 查看 / 重置性能分析器（各阶段调用次数、平均 / 最大 / 总耗时） |

排错用法见 [诊断排错](./troubleshooting.md)。

## 帮助菜单包含哪些条目

`/qr help` 输出的条目（来自语言文件 `commands.yml` 的 `cmd.help-*`）覆盖：help / list / near / activate / reroll / reward / party（玩家段），以及 pos / save / import / spawn / why / stage / genstruct / editor / markers / guide / tp / remove / info / profile / reload（管理段）。
