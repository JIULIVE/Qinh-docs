# config.yml 全配置

> 所属：[参考手册](./commands.md)　·　相关：[自然生成与预加载](../02-server-guide/generation-preloading.md) · [秘境与钥石](../02-server-guide/realms-keystones.md) · [向导与图鉴](../02-server-guide/guide-codex.md)　·　返回：[首页](../README.md)

`plugins/QinhRuins/config.yml` 全字段速查。改后 `/qr reload` 生效。本页是**字段参考**，深入玩法说明链接到对应服主指南。逐字段核对自源码内置 `config.yml`。

> 所有 YAML 路径默认相对 `plugins/QinhRuins/`。玩家可见文案不在此文件，走 `lang/<语言>/*.yml`。

---

## 顶层

| 字段 | 默认 | 说明 |
|---|---|---|
| `config-version` | `1` | 配置版本基线，**勿手动改**（未来结构升级时用于自动迁移） |
| `language` | `zh_cn` | 语言文件夹名，7 种内置之一或自建。缺失 key 回退 `en_US`。见 [内置内容清单](./bundled-content.md#内置语言7-种) |

## storage — 存储后端

```yaml
storage:
  type: yaml      # yaml | database
```

| 值 | 含义 |
|---|---|
| `yaml`（默认） | 单机够用，零配置 |
| `database` | 复用 QinhCoreLib 的数据库（SQLite / MySQL 在 QCL 的 config.yml 配，QR 不单独配库）。旧值 `sqlite`/`mysql` 也按 `database` 处理 |

> 仅**玩家图鉴发现**会进数据库（群组服配同一 MySQL 即跨服共享）；锚点 / 宝箱领取 / 老虎机等绑世界坐标的数据始终本地 YAML。详见 [数据存储](../04-developer/data-storage.md)。

## guide — 指引系统

右键指引物 → 确认 GUI → 扣 1 个并开启罗盘指引。玩法详见 [向导与图鉴](../02-server-guide/guide-codex.md)。

| 字段 | 默认 | 说明 |
|---|---|---|
| `display` | `title` | 显示方式：`title`（屏幕中上方标题）/ `actionbar`（物品栏上方）/ `none`（交给 HUD 插件，配合占位符） |
| `title` | `§e◈ {ruin}` | title 模式主标题（占位符 `{ruin}` `{distance}` `{direction}` `{world}`） |
| `subtitle` | `§f{distance}m §7{direction}` | title 模式副标题 |
| `actionbar` | `§e{ruin} §f{distance}m §7{direction}` | actionbar 模式文本 |
| `not-found` | — | 途中目标临时解析不到时的提示 |
| `other-world` | — | 目标在其它世界时的提示 |
| `search-fail` | — | 开启指引时范围内无目标的提示 |
| `search-radius` | `10000` | 指引搜索半径（格）。太大搜图卡服，按需调 |
| `discover-radius` | `48` | 进入此半径=到达：永久扣除指引物、结束指引 |
| `refresh-ticks` | `10` | 距离 / 方向刷新间隔 tick（越大越省） |
| `exit-action` | `swap` | 游戏内退出指引的动作：`swap`（换手 F）/ `drop`（丢弃 Q）/ `none`。想用任意键：客户端把该键绑到 `/qr guide cancel` |

### guide.confirm — 右键确认 GUI

| 字段 | 默认 | 说明 |
|---|---|---|
| `title` | `Start guidance?` | 确认 GUI 标题（可换 ItemsAdder `:xxx:` / CraftEngine `<image:xxx>` 套图） |
| `yes-name` | `§a✔ Confirm` | 确认按钮名 |
| `no-name` | `§c✘ Cancel` | 取消按钮名 |

### guide.particle — 方向粒子

| 字段 | 默认 | 说明 |
|---|---|---|
| `enabled` | `true` | 是否画方向粒子 |
| `type` | `SOUL_FIRE_FLAME` | 粒子类型（任意原版粒子名；`DUST` 用下方颜色） |
| `points` | `6` | 方向粒子点数（越少越省） |
| `size` | `1.0` | 粒子大小（仅 `DUST`） |
| `min-distance` | `2` | 小于此距离不再画 |
| `near-distance` | `32` | 小于此距离用 `near-color`（仅 `DUST`） |
| `near-color` | `GOLD` | 接近时颜色（仅 `DUST`） |
| `far-color` | `AQUA` | 远离时颜色（仅 `DUST`） |

> HUD 占位符（需 PlaceholderAPI）：`%qinhruins_guide_active%` / `_ruin` / `_distance` / `_direction` / `_world`。见 [占位符](../04-developer/placeholders.md)。

## realm — 秘境（词缀 × 层数 endgame）

玩法详见 [秘境与钥石](../02-server-guide/realms-keystones.md) · [词缀系统](../02-server-guide/affixes.md)。

### realm.tiers — 层数曲线（表达式变量 `tier`）

| 字段 | 默认 | 说明 |
|---|---|---|
| `max` | `16` | 最高层数 |
| `affix-count` | `1 + tier / 3` | 每层词缀数量（表达式） |
| `mob-level` | `tier * 5` | 怪物等级曲线 |
| `loot-bonus` | `1 + tier * 0.1` | 净化战利品数量倍率（按层缩放，与贪婪叠乘；写 `1` 关闭按层加成） |
| `danger-budget` | `40 + tier * 10` | 危险预算上限（词缀 danger 之和封顶，防低层地狱叠加；≤0=不限。首个词缀始终保底） |

### realm.keystone — 钥石外观

右键钥石=激活秘境（已屏蔽末影眼飞天等原版行为）。

| 字段 | 默认 | 说明 |
|---|---|---|
| `material` | `minecraft:ender_eye` | 钥石材质，支持任意物品源（`craftengine:`/`itemsadder:`/`nexo:`/`qinhitems:`/`mmoitems:类型:id`/`neigeitems:`/`mythicmobs:`） |
| `name` | `&dRealm Keystone &7T{tier}` | 名称（`{tier}`=层数；**仅作用于原版材质**；指向其它物品源时保留其自带 name/lore；留空 `""` 则原版材质也不改名） |
| `lore` | （一行） | Lore（同 name 规则） |
| `model-data` | `0` | CustomModelData |

> 自定义物品钥石想显示层数：在它自己的 name/lore 写 `%qinhruins_keystone_tier%`（需 PlaceholderAPI）。

### realm.reward — 净化奖励

钥石天梯外的独占战利品，按参与者私发。

| 字段 | 默认 | 说明 |
|---|---|---|
| `loot-table` | `realm` | 独占战利品表名（`loottables/<名>.yml`） |
| `per-tier-table` | `false` | true 时优先用 `realm_t{tier}`，缺则回退 `loot-table` |
| `growth-scaled` | `true` | 个人掉落数量按成长度缩放 |
| `currency-base` | `tier * 100` | 每位参与者基础货币（表达式，`0`=关闭，需经济插件） |
| `currency-provider` | `""` | 经济后端：留空=auto / `vault` / `excellenteconomy`(ee) / `playerpoints` |
| `currency-id` | `""` | 货币 ID（ExcellentEconomy 必填，如 money/silver/gold；Vault/PP 留空） |

### realm.reroll — 钥石重铸 / 赌博

花货币重 roll 钥石词缀，满意再激活。

| 字段 | 默认 | 说明 |
|---|---|---|
| `enabled` | `true` | 关闭则 `/qr reroll` 不可用，词缀仅激活瞬间盲盒 roll |
| `cost-base` | `tier * 50` | 单次重铸花费（表达式，需经济插件） |
| `currency-provider` | `""` | 同 reward |
| `currency-id` | `""` | 同 reward |

### realm 其它

| 字段 | 默认 | 说明 |
|---|---|---|
| `failure.consume-keystone` | `false` | 失败是否消耗钥石（可按层覆盖；高层建议 true） |
| `activate-radius` | `48` | `/qr activate` 搜索附近遗迹的半径 |
| `abandon-seconds` | `120` | 秘境周围无玩家多久判定放弃（清怪重置，防持久怪堆积） |
| `time-limit` | `"0"` | 秘境限时（秒），`0`=无限时（更像世界结构） |
| `keystone-chance` | `0.5` | 净化后掉同层钥石概率（直接进背包，0~1） |
| `next-tier-chance` | `0.2` | 净化后额外掉更高一层钥石概率（天梯发动机） |
| `bossbar-color` | `PURPLE` | 秘境 bossbar 颜色 |
| `bossbar-style` | `SEGMENTED_10` | 秘境 bossbar 样式 |

| 字段 | 默认 | 说明 |

## vessel — 容器战利品

结构里摆原版容器即自动生效，每人独立滚奖励。玩法见 [战利品系统](../02-server-guide/loot-tables.md)。

| 字段 | 默认 | 说明 |
|---|---|---|
| `enabled` | `true` | 总开关 |
| `refresh-mode` | `per-player-once` | `per-player-once`（每人仅一次）/ `timed`（冷却后可再开重滚） |
| `refresh-cooldown` | `30m` | `timed` 模式冷却（`30m`/`1h`/…） |
| `rows` | `3` | 容器界面行数（1-6） |

> 容器归属由结构占地范围自动判定（无需配半径）；用哪张表在模板 `loot.container-table` 指定。

## protection — 遗迹保护

| 字段 | 默认 | 说明 |
|---|---|---|
| `enabled` | `true` | 探索 / 激活中不可破坏放置，通关后解锁。管理员（`qinhruins.admin`）与「挖穿」机关触发块不受限 |

详见 [刷怪与通关](../02-server-guide/spawning-clearing.md)。

## cleanup — 移除 / 清理

| 字段 | 默认 | 说明 |
|---|---|---|
| `snapshot-restore` | `true` | 生成结构前快照原地形，`/qr remove` / 消退时还原（关闭省磁盘，但移除不还原方块） |
| `max-snapshot-volume` | `500000` | 快照体积上限（方块数）：超此值跳过快照（移除不还原但不卡服）。`0`=不限（危险） |

## staging — 预览暂存

`/qr stage` 放置前用幽灵方块预览。

| 字段 | 默认 | 说明 |
|---|---|---|
| `preview-cap` | `12000` | 幽灵方块预览上限：结构超此值则只画粒子边框。调大可预览更大结构 |

## generation — 世界生成

玩法详见 [自然生成与预加载](../02-server-guide/generation-preloading.md)。

| 字段 | 默认 | 说明 |
|---|---|---|
| `enabled` | `true` | 自然生成总开关（关闭则只能 `/qr spawn` 手动投放） |
| `grid-bucket-size` | `512` | 空间索引分桶尺寸 |
| `max-per-tick` | `4` | 每次处理的候选区块上限（限流） |
| `max-millis-per-tick` | `3` | 每次处理的毫秒预算：超此耗时即让出本 tick（抗压防尖峰） |
| `period-ticks` | `5` | 候选处理任务间隔 tick |
| `spread-place-threshold` | `30000` | 结构方块数超此值且 NONE 朝向→分帧放置（8192 块/tick，不冻服）。小结构仍单发 |
| `marker-blocks` | `true` | 原理图标记块自动处理：屏障→放置时清为空气；基岩→保留原地形不覆盖（地形透出标记）。`false`=原样放置 |

### generation.director — 生成总控

| 字段 | 默认 | 说明 |
|---|---|---|
| `enabled` | `true` | 开启后总控接管：所有遗迹按权重竞争抽签、受全局密度约束。**关闭则回退「每模板各自按 probability 独立 roll」的旧行为** |
| `notify-admins` | `true` | 任意遗迹自然生成时私聊在线管理员报坐标（含 `/qr tp`）。调试找遗迹神器 |
| `chunk-chance` | `80` | 新区块尝试生成的概率分母（每区块 1/N；`0`=探索不主动刷，纯靠 live） |

#### generation.director.live — 活世界

已探索世界也随时间冒出新遗迹。

| 字段 | 默认 | 说明 |
|---|---|---|
| `enabled` | `true` | 活世界开关 |
| `interval` | `10m` | 每隔多久围绕在线玩家尝试一次（刷新间隔，调试可 `1m`/`30s`） |
| `chance` | `0.25` | 每名玩家每次尝试的概率 |
| `min-radius` | `80` | 选址环带内径（避开脚下） |
| `max-radius` | `220` | 选址环带外径（略大于视距） |

#### generation.director.density — 全局密度

| 字段 | 默认 | 说明 |
|---|---|---|
| `region-size` | `512` | 密度统计方格边长 |
| `max-per-region` | `3` | 每方格最多几个遗迹（`0`=不限） |
| `min-spacing` | `48` | 任意两遗迹最小间距（`0`=不限） |

#### generation.director 其它

| 字段 | 默认 | 说明 |
|---|---|---|
| `regen-hours` | `24` | 打通遗迹多久后消退（附近无人时快照还原、腾名额；`0`=永不消退；真消退需开 `cleanup.snapshot-restore`） |
| `weights` | `[]` | 遗迹权重抽签表（`- { template: id, weight: 30 }`）；留空=自动用各模板 probability 折算 |

## 其它运行时数据文件

config.yml 之外，QR 还从这些文件加载内容，见 [资源文件总览](./resource-files.md)：

| 路径 | 内容 |
|---|---|
| `generators/*.yml` | 放置档案（17 个内置） |

| `palettes/*.yml` | 程序化拼接调色板 |
| `loottables/*.yml` | 战利品表（default / vault / realm） |
| `affixes.yml` | 词缀库（13 个内置启用） |
| `scripts/*.js` | 词缀 JS 脚本 |
| `lang/<语言>/*.yml` | 多语言文案（7 种内置） |
| `schematics/*.schem` | 待 `/qr import` 的 WorldEdit 文件 |
| `anchors.yml` | 锚点实例数据（运行时生成，勿手改） |
