# 内置内容清单

QR 首次启动会向 `plugins/QinhRuins/` 释放下列内置内容。本页逐项列出（按源文件核对），便于按需复制 / 改名 / 删改。

---

## 一、内置放置档案（`generators/`，17 个）

放置档案=「在哪生成」的可复用规则包（维度 / 群系过滤、生成层、平整度、间距、概率）。模板用 `generation.profile: <名>` 引用，可局部覆盖任意字段。详见 [放置档案](../02-server-guide/placement-profiles.md)。

> 命名规律 `<生成层>_<群系/维度>`。`layer` 取值 `surface`（地表）/ `underground`（地下）/ `sky`（天空）/ `ocean-surface`（海面）/ `seabed`（海底）。`y-band` 是地下 / 天空的随机深度 / 高度。

### 主世界（NORMAL，11 个）

| 文件 | 层 | 群系 / 说明 |
|---|---|---|
| `surface_overworld.yml` | surface | 地表**通用**，不限群系（其它地表档案的基底，分母 700） |
| `surface_grassland.yml` | surface | 草木带：平原 / 各类森林 / 丛林 / 沼泽 / 针叶林 / 高山草甸（17 群系） |
| `surface_desert.yml` | surface | 荒漠：仅 `DESERT`，落地白名单 SAND / SANDSTONE / RED_SAND |
| `surface_snowy.yml` | surface | 冰雪带：雪原 / 雪林 / 冰刺 / 冰封山峰 / 雪坡（8 群系） |
| `surface_barren.yml` | surface | 荒原：恶地 / 热带草原 / 蘑菇岛 / 裸岩山峰（8 群系） |
| `underground_overworld.yml` | underground | 地下通用·埋藏地宫 / 墓穴，随机埋深 y 10~35 |
| `underground_lush.yml` | underground | 繁茂洞穴：仅 `LUSH_CAVES`，y 6~24 |
| `underground_dripstone.yml` | underground | 滴水石洞穴：仅 `DRIPSTONE_CAVES`，y 8~30 |
| `sky_overworld.yml` | sky | 悬浮空岛 / 浮空遗迹，悬浮高度 y 35~75 |
| `ocean_overworld.yml` | ocean-surface | 海面·沉船 / 浮台，坐落海平面（无水自动换点，9 类海洋群系） |
| `seabed_overworld.yml` | seabed | 海底·沉到海床（结构须自带封闭外壳，否则内部灌水） |

### 下界（NETHER，3 个）

| 文件 | 层 | 说明 |
|---|---|---|
| `surface_nether.yml` | surface | 下界地表：坐落地形顶面，已避开基岩天花板（可按 NETHER_WASTES / CRIMSON_FOREST 等细分） |
| `underground_nether.yml` | underground | 下界地下：埋在地狱岩层里，y 8~24 |
| `sky_nether.yml` | sky | 下界空中：悬浮在熔岩海上方（y 18~38，下界顶是基岩，别配太高） |

### 末地（THE_END，3 个）

| 文件 | 层 | 说明 |
|---|---|---|
| `surface_end.yml` | surface | 末地地表：坐落末地岛屿表面（可按 END_HIGHLANDS / END_MIDLANDS 等细分） |
| `underground_end.yml` | underground | 末地地下：埋入末地石岛内部，y 6~18 |
| `sky_end.yml` | sky | 末地虚空浮岛：悬浮在末地岛屿上方，y 25~60 |

## 二、内置战利品表（`loottables/`，3 张）

战利品表=可复用的掉落配置（`rolls` 滚动次数 + 加权 `entries` + 可选条件分组）。`item` 走 QinhCoreLib 的 `ItemManagerAPI` 解析，支持 `vanilla:`（原版）/ QI / NeigeItems / ItemsAdder / CraftEngine / Nexo 等任意来源。详见 [战利品系统](../02-server-guide/loot-tables.md)。

| 表 | 文件 | 用途 |
|---|---|---|
| **default** | `default.yml` | 编辑器标记宝箱的默认表（兜底，避免「奖励表缺失」）。`rolls: 2`，原版铁 / 金 / 钻 / 经验瓶 / 面包 |
| **vault** | `vault.yml` | 容器战利品 + 通关大奖通用。`containers: [CHEST, BARREL]` 限定生效容器；含 `groups` 条件分组（`condition` 走 PlaceholderAPI，无 PAPI 时恒真）；注释里给出叠加原版战利品表的写法 |
| **realm** | `realm.yml` | 秘境净化独占奖励表（按参与者私发）。`rolls: 3`，条目可带 `min-growth`（成长度门槛）与 `unique: true`（受「秘藏」词缀加权）。如下界合金碎片需成长度 ≥30、附魔金苹果需 ≥50 |

> 在模板 `template.yml`：`loot.container-table: vault` = 容器战利品；`reward.clear-table: vault` = 通关老虎机奖池；`realm` 表通常配在秘境净化产出。

## 三、内置词缀（`affixes.yml`，13 个）

词缀=秘境 强化修饰。每个词缀有 `category`（MOB/PLAYER/ENV/LOOT）、可选 `group`（同组互斥）、`danger`（吃[危险预算](./glossary.md)）、`reward`、`min-tier`、`effect`。详见 [词缀系统](../02-server-guide/affixes.md)。

> 🔑 **架构红线**：QR 不算属性 / 伤害。「怪变强」靠 `mob-level-bonus` 委托 **MythicMobs 提 level** 自行缩放 stat，不是 QR 改数值。

| 词缀 id | 显示名 | 类别 | 效果（effect.type）| 互斥组 / 门槛 |
|---|---|---|---|---|
| `elite-swarm` | 精英成群 | MOB | 刷怪数量 ×1.5（`mob-count-mult`） | group mob-count，min-tier 2 |
| `horde` | 怪潮 | MOB | 刷怪数量 ×2.0 | group mob-count，min-tier 5 |
| `veteran` | 百战 | MOB | 怪物等级 +2（`mob-level-bonus`，MM 缩放） | group mob-level，min-tier 1 |
| `overlord` | 霸主 | MOB | 怪物等级 +5（MM 缩放） | group mob-level，min-tier 4 |
| `oppression` | 重压 | PLAYER | 区域缓慢 I + 挖掘疲劳 I（`player-status`） | min-tier 1 |
| `enfeeble` | 虚弱诅咒 | PLAYER | 区域虚弱 I（`player-status`） | min-tier 2 |
| `eternal-night` | 永夜 | ENV | 区域黑暗（`player-status` DARKNESS） | min-tier 1 |
| `brittle-ward` | 脆弱结界 | PLAYER | 禁用一切治疗（`no-heal`） | min-tier 5，danger 45 |
| `sands-of-time` | 时之沙 | ENV | 限时 5 分钟内净化（`time-limit` 300s） | min-tier 3 |
| `greed` | 贪婪 | LOOT | 净化掉落数量 ×2.5（`loot-quantity-mult`） | danger 0，min-tier 1 |
| `golden-touch` | 鎏金 | LOOT | 货币掉落 ×3.0（`currency-mult`，走经济桥） | danger 0，min-tier 1 |
| `hidden-trove` | 秘藏 | LOOT | 独占条目（`unique:true`）权重 ×3（`unique-chance-mult`） | danger 0，min-tier 3 |
| `herald` | 秘境降临 | ENV | 激活时全服播报（`broadcast`，message 支持 `{ruin}`/`{tier}`） | danger 0，min-tier 1 |

> `affixes.yml` 末尾还有两段**注释掉的扩展示例**（去掉 `#` 即启用）：`blood-ritual`（命令型 `type: command`，对在场玩家执行命令，占位 `{player}`/`{tier}`）与 `scripted-trial`（JS 脚本型 `type: script`，逻辑在 `scripts/affix_example.js`，引用语法 `qinhruins:文件.js:函数`）。

## 四、内置语言（`lang/`，7 种）

玩家可见文案全部走 `lang/<语言>/*.yml`，由 `config.yml` 的语言项选定，缺失 key 自动回退 `en_US`。每种语言是一个文件夹，内含多个 `.yml`（loader 自动合并）。

| 文件夹 | 语言 | 翻译状态 |
|---|---|---|
| `zh_cn` | 简体中文 | 完整（默认） |
| `en_US` | 英语 | 完整（回退基准） |
| `zh_tw` | 繁体中文 | 提供 |
| `ru_RU` | 俄语 | 提供 |
| `fr_FR` | 法语 | 提供 |
| `vi_VN` | 越南语 | 提供 |
| `es_ES` | 西班牙语 | 提供 |

> 加新语言：复制 `lang/en_US/` 整个文件夹改名，翻译里面各 `.yml`，再把 config 的语言项改为新文件夹名。

## 五、示例模板（`templates/_example/`）

| 文件 | 说明 |
|---|---|
| `templates/_example/template.yml` | 遗迹模板**全字段注解**参考（结构 / 掩码 / 标记块 / generation / loot 等逐字段注释） |
| `templates/_example/blueprint.yml` | 蓝图玩法层示例：刷怪点 / 奖励箱 / 击杀目标 / 核心 / 机关 / 子结构变体 |

> `_` 开头的文件夹会被加载器**忽略**、不当真遗迹注册——它是只读参考样板。要造真遗迹，复制一份去掉下划线，或 `/qr save <id>` / 用[可视化编辑器](../03-editor/editor-overview.md)。详见 [遗迹模板](../02-server-guide/ruin-template.md)、[蓝图与目标](../02-server-guide/blueprint-objectives.md)。

## 六、内置脚本（`scripts/`）

| 文件 | 说明 |
|---|---|
| `scripts/affix_example.js` | JS 词缀脚本示例，供 `type: script` 的词缀引用（`qinhruins:affix_example.js:onActivate`）。`onActivate(ctx)` 在秘境激活瞬间对在场每位玩家各执行一次，提供 `ctx.player()` / `ctx.get("tier")` / `ctx.itemGive` / `ctx.addPotion` / `ctx.economyDeposit` / `qcl.logInfo` 等接口。走 QinhCoreLib 的 GraalJS。详见 [词缀脚本](../04-developer/affix-scripting.md) |

## 七、首次释放说明

- 上述内容仅在**首次启动**释放，已存在的文件**不会被覆盖**（你的改动安全）。
- 想恢复某个被删的内置文件：从 QR jar 里单独取，或删除后重启释放。
- 程序化拼接的调色板（Tile Palette）默认**无内置**，需自行编写。详见 [程序化生成](../02-server-guide/procedural-generation.md)。
