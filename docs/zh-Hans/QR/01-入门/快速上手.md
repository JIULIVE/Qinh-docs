# 5 分钟快速上手

> 上一页：[安装](./installation.md)　·　下一页：[核心概念](./core-concepts.md)

本页带你**从零造出第一座会自然生成、能探索、能通关的遗迹**。有两条路：**A 用 GUI 编辑器**（推荐新手）和 **B 手写 YAML**（懂结构后更快）。先走 A。

> 前置：已按 [安装](./installation.md) 装好 QCL + QR，进游戏有 OP 权限。

---

## 路线 A：可视化编辑器（推荐）

### 第 1 步 · 搭一座建筑

在世界里随便搭一座小建筑——一间石头房、一座塔都行。记住它的范围。

### 第 2 步 · 框选并存成模板

站在建筑的一个角，看向**最低角的方块**，输入：

```
/qr pos1          # 看着方块按命令=设第一个角
```

走到对角，看向**最高角的方块**：

```
/qr pos2
/qr save my_ruin  # 把选区结构存成模板 my_ruin
```

这一步会生成 `templates/my_ruin/`，里面有结构文件和一份默认 `template.yml`。

### 第 3 步 · 进编辑器标点

```
/qr editor start my_ruin     # 进入该遗迹的可视化编辑器
```

进入后你会拿到编辑工具。按提示：

- `/qr editor mode spawn` → 站在某处把它标为**刷怪点**，再 `/qr editor set mob ZOMBIE`、`set count 3`、`set stage 1`。
- `/qr editor mode chest` → 标一个**奖励箱**，`set loot vault`、`set unlock 1`（阶段 1 通关才解锁）。
- `/qr editor mode core` → 标**遗迹核心**（钥石激活秘境用）。
- `/qr editor save` → 存盘，自动写好 `blueprint.yml`。

> 编辑器全流程见 [可视化建迹](../03-editor/visual-building.md)。

### 第 4 步 · 让它自然生成

打开 `templates/my_ruin/template.yml`，确认：

```yaml
generation:
  profile: surface_overworld   # 用内置「地表·主世界」放置档案
  enabled: true                # 参与自然生成
  weight: 20                   # 抽签权重
```

`/qr reload` 重载。现在它会在主世界地表自然冒出。

### 第 5 步 · 找到它、验证它

不想等自然生成？直接手动投放或调试：

```
/qr spawn my_ruin     # 在你脚下立刻生成一座
/qr gentest           # 在附近强制生成一座（绕过概率，调试用）
/qr near 500          # 列出附近 500 格内的遗迹
/qr tp <锚点id>       # 传送到某座遗迹
```

走近它——怪物会激活、宝箱可开、击杀目标开始计数。全清目标 → 通关、解锁奖励。🎉

## 路线 B：手写 YAML（懂结构后更快）

如果你已有 `.schem` 原理图（WorldEdit / 下载的）：

```
# 1. 把 xxx.schem 丢进 plugins/QinhRuins/schematics/
/qr import xxx my_ruin    # 转成 templates/my_ruin/，原生 .nbt + 模板骨架
```

然后手写 `templates/my_ruin/blueprint.yml`（坐标都相对结构原点 0,0,0）：

```yaml
spawn-points:
  - { id: s1, x: 5, y: 2, z: 5, mob: ZOMBIE, count: 3, level: 1, stage: 1 }
loot-chests:
  - { id: c1, x: 3, y: 2, z: 4, loot-table: vault, unlock-stage: 1, per-player-once: true }
objectives:
  stages:
    - { stage: 1, name: "清剿", kills: { ZOMBIE: 3 } }
```

`/qr reload` 即可。模板字段全解见 [遗迹模板](../02-server-guide/ruin-template.md)，蓝图字段全解见 [蓝图与目标](../02-server-guide/blueprint-objectives.md)。

## 调试三件套（遗迹不生成时必用）

| 命令 | 作用 |
|---|---|
| `/qr why <模板>` | 站在某处问「这里为什么生成 / 不生成这个遗迹」，逐条列出选址检查（✔/✘） |
| `/qr gentest` | 在附近强制生成一座（绕过概率），验证结构 / 蓝图本身没问题 |
| `/qr spawn <模板>` | 在脚下精确投放一座 |

> 遗迹「死活不出」的系统排查见 [诊断排错](../05-reference/troubleshooting.md)。
