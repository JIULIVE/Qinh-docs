# 安装与环境

> 上一页：[概览](./overview.md)　·　下一页：[快速上手](./quick-start.md)

## 环境要求

| 项 | 要求 | 说明 |
|---|---|---|
| 服务端 | Paper / Purpur / Spigot **1.21.11+** | QR 用 Bukkit **原生结构 API** 放置结构，需要较新内核 |
| Java | **25+** | 与秦淮全家桶一致 |
| 硬依赖 | **QinhCoreLib** | 必须先装；没装 QR **直接不启用** |

启动时 QR 会自检：QCL 未加载 → 控制台报错并禁用自身；服务端版本不符 → 同样禁用。看到日志里那串 `QinhRuins x.xx.x` logo banner，就说明启用成功。

---

## 安装步骤

1. 先把 **QinhCoreLib** 放进 `plugins/`（它是全家桶底座）。
2. 把 **QinhRuins** 的 jar 放进 `plugins/`。
3. （可选）装上你需要的软依赖（见下表）。
4. 启动服务器。首次启动 QR 会在 `plugins/QinhRuins/` 释放一整套默认资源：
   - `config.yml` —— 主配置
   - `templates/_example/` —— 带注释的遗迹模板示例（`_` 开头会被忽略，不当真遗迹加载）
   - `generators/*.yml` —— 17 个内置放置档案
   - `loottables/*.yml` —— `default` / `vault` / `realm` 三张示例战利品表
   - `affixes.yml` —— 词缀库
   - `scripts/affix_example.js` —— 词缀脚本示例
   - `lang/<语言>/*.yml` —— 7 种语言文案
5. 进游戏 `/qr version` 确认版本，`/qr` 看帮助。

::: warning 注意
首次启动后**先别急着删 `_example`**——它是你照着改的最佳模板。
:::

## 软依赖与降级表

QR 的后端**全部自动探测、无需手动配置**。缺哪个软依赖，对应能力**优雅降级**，插件照常启动：

| 软依赖 | 提供能力 | 缺了会怎样（降级） |
|---|---|---|
| **MythicMobs** | 刷自定义怪（蓝图里 `mm-怪名`） | `mm-` 前缀的怪刷不出；原版实体名照常 |
| **QinhClass / MMOCore** | 玩家**成长度**、**队伍**来源 | 成长度缩放退化为中性；队伍退化为 QR **内置临时队伍**（`/qr party`） |
| **QinhItems / NeigeItems / MMOItems** | 战利品 / 钥石 / 指引物引用自定义物品 | 这些来源的 `item:` 解析失败被跳过；`vanilla:` 原版物品照常 |
| **ItemsAdder / CraftEngine / Nexo** | 钥石 / 指引物用资源包套图 | 同上，回退原版材质 |
| **PlaceholderAPI** | 全部 `%qinhruins_*%` 占位符、战利品表 `condition` 占位符 | 占位符不解析；战利品 `condition` 视为恒真 |
| **Vault / ExcellentEconomy / PlayerPoints** | 秘境货币奖励、进入花费、钥石重铸花费 | 货币相关功能静默跳过 |
| **Citizens** | 机关 `NPC` 动作生成 NPC | `NPC` 动作不生效 |

> 经济后端的选择见 [配置文件](../05-reference/config.md) 的 `realm.reward.currency-provider`；留空 = 自动探测。

## 存储后端

QR 默认用 **YAML**（单机零配置即用）。`config.yml`：

```yaml
storage:
  type: yaml      # yaml | database
```

- **`yaml`** —— 锚点、宝箱领取、老虎机、图鉴等全部存本地 YAML 文件。单机 / 单服够用。
- **`database`** —— **复用 QinhCoreLib 的数据库**（SQLite 还是 MySQL 在 QCL 的 `config.yml` 里配，QR 不再单独配库）。

> 群组服（BungeeCord / Velocity）想让**玩家图鉴发现跨服共享** → 把 QinhCoreLib 配成同一个 MySQL，各后端自然共享。
>
> 注意：**只有图鉴（per 玩家）会进数据库**；锚点 / 宝箱领取 / 老虎机等绑具体世界坐标的数据仍各服本地 YAML（本就该本地）。详见 [数据存储](../04-developer/data-storage.md)。

## 升级与配置迁移

QR 内置**配置迁移器**：`config.yml` 顶部有 `config-version` 基线，**勿手动改**。未来配置结构升级时，迁移器会在启动时自动补齐新字段、保留你的旧值——老服直接换 jar 即可，不用重置配置。
