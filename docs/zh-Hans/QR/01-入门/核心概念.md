# 核心概念与架构

> 上一页：[快速上手](./quick-start.md)　·　返回：[首页](../README.md)

本页把 QR 的每个名词讲清楚，并画出它们怎么串成一条流水线。**读完这页，后面所有服主 / 开发文档你都能秒懂。**

---

## 一张图看懂全貌

```text
              ┌─────────── 你提供的「配方」（磁盘上的 YAML + 结构文件）───────────┐
              │                                                               │
  结构文件 ──→ 模板(Template) ──引用──→ 放置档案(Profile)      蓝图(Blueprint)   │
  .nbt        template.yml          generators/*.yml       blueprint.yml      │
              │                          │                      │             │
              └──────────────────────────┼──────────────────────┘             │
                                         ▼                                     ▼
                            生成总控抽签选址、粘结构、融地形
                                         │
                                         ▼
                              锚点(Anchor)  ← 世界里一座真实存在的遗迹实例
                                         │
                          ┌──────────────┼───────────────┐
                          ▼              ▼               ▼
                   怪物激活/目标      机关(Mechanism)    战利品(Loot)
                          │
                          ▼  通关后可被钥石(Keystone)注入
                              秘境(Realm) = 锚点 + 层数 + 词缀(Affix)
```

> 一句话串起来：**「模板 + 放置档案 + 蓝图」是配方（磁盘上），「锚点」是按配方在世界里生成的成品实例，「秘境」是给成品叠 buff 的反复刷玩法。**

---

## 配方层（磁盘上的定义，可复用）

### 模板（Template）— `template.yml`

一座遗迹的**身份证**。它回答：「这是什么遗迹、用哪个结构文件、长啥样、在哪生成、出什么战利品、进入门槛是什么」。每个模板是 `templates/<id>/` 一个文件夹，里面有：

- `template.yml` —— 模板定义本体
- 结构文件（默认 `structure.nbt`）—— 遗迹的方块本体
- `blueprint.yml` —— 玩法层（刷怪点 / 目标 / 机关 / 核心）

`_` 开头的文件夹（如 `_example`）会被忽略，不当真遗迹加载。模板全字段见 [遗迹模板](../02-server-guide/ruin-template.md)。

### 结构文件（Structure）

遗迹的**方块本体**。QR 用 Bukkit **原生 `.nbt`**；WorldEdit 的 `.schem` 用 `/qr import` 转成 `.nbt`。放置时支持掩码融合、地基填充、标记块、分帧防卡。详见 [结构文件](../02-server-guide/structure-files.md)。

### 放置档案（Profile）— `generators/*.yml`

**「在哪生成」的可复用规则包**：维度 / 世界 / 群系过滤、生成层（地表 / 地下 / 天空 / 海面 / 海底）、平整度、最小间距、概率…… 多个模板可以共享同一个档案（`generation.profile: surface_overworld`），也可在模板里**局部覆盖**任意字段。内置 17 个。详见 [放置档案](../02-server-guide/placement-profiles.md)。

### 蓝图（Blueprint）— `blueprint.yml`

叠在结构上的**玩法层**。所有坐标都是**相对结构原点**（结构最小角 = 0,0,0）的方块坐标：

- **刷怪点（spawn-points）** —— 在哪刷什么怪、几只、几级、属于哪个阶段
- **击杀目标（objectives）** —— 分阶段的通关条件
- **奖励箱（loot-chests）** —— 解锁式宝箱
- **核心（cores）** —— 钥石激活秘境的交互点
- **机关（mechanisms）** —— 触发器 → 动作
- **子结构变体（variants）** —— 槽位上贴加权随机模板

详见 [蓝图与目标](../02-server-guide/blueprint-objectives.md)。

## 实例层（世界里真实存在的东西）

### 锚点（Anchor）

**一座按模板在世界某坐标生成出来的真实遗迹**，就是「实例」。锚点记录：用哪个模板、在哪、朝向、尺寸（宽 / 高 / 深）、当前状态。所有运行时玩法（激活刷怪、目标计数、机关、保护、战利品归属）都挂在锚点上。

锚点有 **4 个生命周期状态**：

| 状态 | 含义 |
|---|---|
| `DORMANT` | 已生成但无人靠近，怪物 / 机关休眠 |
| `ACTIVE` | 有玩家进入激活半径，怪物激活、目标计数中 |
| `CLEARED` | 通关 / 已被探索完，宝箱开过、保护解除 |
| `RECYCLED` | 已被回收（消退还原或 `/qr remove`），名额已腾出 |

> 模板 vs 锚点：**一个模板可以生成出很多锚点**（世界各处的同款遗迹），就像一个类可以 new 出很多对象。

### 生成总控

负责把配方变成锚点的**调度大脑**：所有可生成的遗迹按**权重竞争抽签**，受**全局密度**（每区域上限、最小间距）约束，还能在**已探索的世界**里定时围绕玩家刷新（活世界），以及让**打通的遗迹随时间消退再生**。关掉生成总控则回退到「每个模板各自按概率独立 roll」的旧行为。详见 [自然生成与预加载](../02-server-guide/generation-preloading.md)。

## endgame 层（给成品叠 buff 反复刷）

### 秘境（Realm）

把一座**通关的遗迹**注入「**层数（tier）+ 词缀（affix）**」后的强化态。层数越高，怪越强、词缀越多、战利品越肥。秘境**不是新世界**，还是原地那座遗迹——只是怪重刷、规则改写、产出升级。详见 [秘境与钥石](../02-server-guide/realms-keystones.md)。

### 钥石（Keystone）

**激活秘境的钥匙**。玩家手持钥石右键遗迹核心 → 该遗迹升为对应层数的秘境。钥石可由通关掉落（带天梯：有概率掉更高一层），也可重铸（花货币重 roll 词缀）。

### 词缀（Affix）

秘境的**强化修饰**，分五类：**数量**（刷怪 ×N）、**等级**（怪 level +N，由怪物插件缩放）、**环境 / 规则**（禁疗 / 限时 / 黑暗）、**玩家状态**（缓慢 / 虚弱）、**战利品**（贪婪 / 鎏金 / 秘藏）。受**危险预算**约束防地狱叠加，同组互斥。

> 🔑 **架构红线**：QR **不算属性 / 伤害**。词缀让「怪变强」是**委托 MythicMobs 提 level**（MM 自己缩放 stat），不是 QR 去改数值。这条和 QI / QS「不碰数值」是同一种思路。

详见 [词缀系统](../02-server-guide/affixes.md)。

## 横切系统（贯穿各层）

| 系统 | 一句话 | 详见 |
|---|---|---|
| **机关（Mechanism）** | 蓝图里的「触发器 → 动作」，纯世界交互（开门 / 刷伏兵 / 传送），非属性 | [机关系统](../02-server-guide/mechanisms.md) |
| **战利品（Loot）** | 容器战利品（每人独立 / 全服共享）+ 解锁奖励箱 + 净化老虎机，成长度缩放 | [战利品系统](../02-server-guide/loot-tables.md) |
| **向导（Guide）** | 指引物右键 → 罗盘指向最近同类遗迹 | [向导与图鉴](../02-server-guide/guide-codex.md) |
| **图鉴（Codex）** | 玩家发现遗迹后记录的「攻略卡」收集册 | [向导与图鉴](../02-server-guide/guide-codex.md) |
| **会话 / 队伍（Session）** | 进入 / 退出提示、bossbar、组队共享进度 | [组队与会话](../02-server-guide/party-sessions.md) |
| **保护（Protection）** | 探索 / 激活中的遗迹不可破坏放置，通关后解锁 | [刷怪与通关](../02-server-guide/spawning-clearing.md) |
| **快照 / 再生（Snapshot）** | 生成前快照原地形，移除 / 消退时还原 | [自然生成与预加载](../02-server-guide/generation-preloading.md) |

## 数据怎么存

- **配方**（模板 / 蓝图 / 档案 / 战利品表 / 词缀）= 磁盘上的 YAML + 结构文件，服主编辑、`/qr reload` 生效。
- **实例与运行态**（锚点、宝箱领取记录、老虎机、机关触发记录）= 本地 YAML（`anchors.yml` 等），绑具体世界坐标，**就该本地**。
- **玩家图鉴发现** = 可选进数据库（跨服共享），其余仍本地。

详见 [数据存储](../04-developer/data-storage.md)。

## 一个完整例子串起所有概念

> 你写了模板 `ancient_tower`（**Template**），它引用放置档案 `surface_overworld`（**Profile**）、配了 `blueprint.yml`（**Blueprint**）里阶段 1 刷 3 僵尸、阶段 2 刷 1 首领。
>
> 玩家探索时，**生成总控** 在某座山谷抽中它、把结构粘进去并融合地形，诞生一个 **Anchor**（状态 `DORMANT`）。玩家走近 → `ACTIVE`，怪激活；踩到红石板触发 **Mechanism** 开门；全清两阶段目标 → `CLEARED`，解锁奖励箱发 **Loot**，并掉落一枚 T3 **Keystone**。
>
> 玩家右键遗迹**核心** → 注入「霸主 + 贪婪 + 禁疗」三个 **Affix**，这座锚点升为 T3 **Realm**；净化后拿独占战利品和更高层钥石。一天后无人靠近，这座打通的遗迹**快照还原**、状态转 `RECYCLED`，世界腾出名额长出新遗迹。

每个加粗词都能在本页找到定义——现在你已经掌握 QR 的全部核心概念了。
