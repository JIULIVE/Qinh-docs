# 📦 服主指南

这是 QinhItems 最大、也是最核心的一节：一份完整的「**如何做一件物品**」服主手册。从用一段 YAML 描述物品的外观，到一层层叠上品质、属性、词缀、套装、宝石孔，再到让服务器自动随机生成装备——做物品要用到的每一个旋钮，都在这里。

整条路线大致是这样：先在 **物品定义** 里学会写一件物品的骨架，再决定它 **属于哪个类型**（类型决定它能用哪些功能）；接着给它穿上 **品质** 外衣、挂上 **属性数值**，用 **变量** 让每件物品滚出不同的数字；想要随机化与可复用，就交给 **词缀**、**段**、**套装** 这套体系；最后用 **随机生成**、**宝石孔**、**灵魂绑定**、**进阶配方** 把玩法做厚，用 **资源包** 给物品换上自定义外观。

可以把这二十页分成五组来记：**示例库**（物品示例库总览 + 三个示例分篇，约 140 件可直接 `/qi give` 领取的内置配方）、**基础结构**（物品定义 / 物品类型 / 品质与显示）、**属性与数值体系**（属性与数值 / 变量引擎 / 词缀 / 段 / 套装）、**进阶玩法**（随机生成 / 宝石孔 / 灵魂绑定 / 进阶配方 / 附魔上限 / 碎片与模板）、**外观**（资源包与模型）。

从哪开始？**新服主**建议先逛 **物品示例库总览**，在游戏里领几件看效果，再对照源码 yml；想从头理解的，按 **物品定义 → 物品类型 → 属性与数值** 顺着读最稳。改完任何 yml 记得 `/qi reload` 才会生效。

## 本节内容

- 📝 [物品定义](./item-definition) — QI 最核心的一章：用 YAML 描述一件物品的全字段与分组文件布局。
- 📚 [物品示例库总览](./item-cookbook) — 约 140 件内置示例的完整索引、逐类注解与即用配方入口。
- 🛡️ [示例·装备篇](./cookbook-equipment) — 护甲 / 武器 / 弓弩 / 盾牌 / 法杖等约 47 件装备示例的逐条 YAML。
- 💍 [示例·饰品/消耗篇](./cookbook-accessories-consumables) — 饰品 / 戒指 / 食物 / 卷轴 / 材料 / 钓竿等约 41 件示例的逐条 YAML。
- 🎲 [示例·杂项篇](./cookbook-misc) — 货币 / 代币 / 奖杯 / 头颅 / 道具 / 坐骑等约 22 件特殊物品示例。
- 🗂️ [物品类型](./item-types) — 类型与能力矩阵：决定一件物品属于哪一类、能用哪些功能。
- 🌈 [品质与显示](./quality-display) — 品质（Tier）的定义与发光、以及最终物品 Lore 的排版规则。
- 📊 [属性与数值](./attributes-numbers) — QI 如何把属性挂上物品并桥接给 AttributePlus 作用到玩家。
- 🔢 [变量引擎](./variables) — 物品级动态值：随机数值、品质词、星级与动态文本占位符。
- 🏷️ [词缀](./affixes) — 可复用的修饰包：给物品加前后缀、追加 Lore，配合池实现随机词缀。
- 📋 [段与 Lore 池](./sections) — 可复用 Lore 块：静态段、加权随机池与品质分支池。
- 🎽 [套装](./sets) — 多件组合穿戴后按件数分级激活属性 / 药水 / 主动技能加成。
- 🎰 [随机物品生成](./random-generation) — 像 MMOItems 一样按品质权重抽 Tier、抽词缀，自动生成装备。
- 💠 [宝石孔](./gem-sockets) — 在装备上开孔，对接 Legendinlay / MagicGem 后端完成镶嵌。
- 🔒 [灵魂绑定](./soulbinding) — 物品绑定到玩家后只有归属者能用、不可丢弃 / 交易 / 死亡掉落。
- 🍳 [进阶配方](./advanced-recipes) — 把变量、段、词缀、套装、动作组合起来的复杂实战配方。
- ✴️ [附魔上限](./enchant-caps) — 限制单件物品的附魔总等级，在铁砧 / 附魔台强制执行。
- 🧱 [碎片与模板](./fragments-templates) — 可复用的物品 YAML 片段，把共享字段抽出一处维护、多处引用。
- 🎨 [资源包与模型](./resource-pack) — 声明自定义模型数据或模型资源路径，配合资源包显示自定义外观。
