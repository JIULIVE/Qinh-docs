# 🎮 编辑器 GUI

QI 提供一整套**游戏内可视化编辑器**，让服主不碰一行 YAML 就能创建和编辑物品、动作、套装。GUI 保存的就是同一份 YAML，和手写完全互通——你随时可以在界面里改、在文件里看，两边互不打架。

入口只有一个命令 `/qi editor`（权限 `qinhitems.editor`）：先进**类型浏览器**，选类型后进**物品浏览器**，右键已有物品或点「新建」即进入**物品编辑器**主界面。物品编辑器是整套 GUI 的中枢，所有子编辑器——动作、宝石孔、属性、套装、模板、资源包——都从这里的字段行点进去。

不同需求对应不同子编辑器：配触发器与处理器去**动作/技能编辑器**；做几件一组的成套加成去**套装编辑器**；管碎片、变量、资源包模型则在**其他子编辑器**里。所有界面都是 6 行 54 槽的统一布局，按钮位置固定。

第一次上手建议先看[操作流程](./workflow)，照着分步点一遍创建武器；想精确知道「点哪个槽」再翻[槽位速查](./slot-reference)。

- 🧭 [概览与导航树](./overview) — 编辑器入口命令、权限，以及从类型浏览器到各子编辑器的完整导航树。
- 🪜 [操作流程](./workflow) — 常见任务（如从零创建一件武器）的分步实战操作流，照着点就能完成。
- 🖊️ [物品编辑器](./item-editor) — 单件物品的编辑主界面：开关、聊天输入、子编辑器三类字段及其写入位置。
- ⚡ [动作/技能编辑器](./action-editor) — 在 GUI 里给物品配触发器、处理器与载荷，绑定技能，无需手写 triggers。
- 🎽 [套装编辑器](./set-editor) — 创建套装、绑定成员件、编排各档加成的套装中心与编辑界面。
- 🧩 [其他子编辑器](./other-editors) — 模板中心、资源包/模型中心，以及编辑器的会话与保存机制。
- 🔢 [槽位速查](./slot-reference) — 各编辑器界面的精确槽位图与通用布局常量，供进阶排错用。
