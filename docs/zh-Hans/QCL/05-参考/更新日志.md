# 更新日志（Changelog）

> 导航：[文档首页](../README.md) · [目录](../README.md) · [FAQ](./faq.md) · [物品源引用](../02-server-guide/item-source-references.md) · [术语表](./glossary.md)

本项目遵循[语义化版本](https://semver.org/lang/zh-CN/)（SemVer）：版本号格式为 `主版本.次版本.修订号`。新增向后兼容的功能时递增**次版本号**，向后兼容的修复时递增**修订号**。

**当前版本：1.3.0**，约 150 个 Kotlin 文件。

---

## [1.3.0] - 2026-06-28

> 兼容范围扩展到 **Minecraft 1.21.11 ~ 26.2.x**（api-version 仍为 1.21，向上兼容，无需改配置）。

### 新增 — 战斗管线（战斗底座大幅强化）

- **槽位 / 来源感知修饰符**：属性修饰符可标记生效槽位（主手 / 副手 / 护甲 / 饰品），战斗时按手过滤。未标记的修饰符行为不变。
- **可插拔伤害来源**：新增 `AttackHandler` 注册表，其它插件可注册自定义伤害来源，让非原版 cause 的伤害也进入分量管线。
- **`DamagePacket` 升级为「计算底座」**：伤害分量可在事件中追加 / 按类型或元素缩放 / 重算（`event.addPacket / scaleType / scaleElement / scaleAll`），消费者不再只能改总标量。
- **限时修饰符折叠缓存**：纯性能优化，战斗读值不再每次重折叠全部限时 buff（输出逐字节一致）。

### 新增 — 激活的战斗属性

- **暴击抗性 `crit_resist`**：降低受到暴击的概率（攻击方暴击率扣减目标该值）。
- **伤害反弹 `reflection`**：受击时反弹一定比例伤害给攻击者。走可编辑脚本 `scripts/attributes/reflection.js`。
- **法术吸血 `spell_vampirism`**：造成法术 / 技能伤害时按比例回血，机制与物理吸血（lifesteal）对称。走可编辑脚本 `scripts/attributes/spell_vampirism.js`，由新增的 `on_magic_damage_dealt` 钩子事件触发。
- **技能伤害隔离**：MM 技能伤害若以 `ENTITY_ATTACK` 落地，借 MythicLib 的伤害类型识别将其从物理近战管线隔离，避免被误套物理增伤 / 暴击（开关 `combat.isolate-mythiclib-skill-damage`）。需装 MythicLib；最佳实践仍是把技能配成 `MAGIC` cause。

### 新增 — 现代 1.21 原版属性

- 接入 18 个 1.21 原版属性（`vanillaKey` 驱动）：`scale`（体型）、`jump_strength`（跳跃力）、`gravity`、`step_height`、`safe_fall_distance`、`fall_damage_multiplier`、`burning_time`、`explosion_knockback_resistance`、`movement_efficiency`、`water_movement_efficiency`、`oxygen_bonus`、`block_interaction_range`、`entity_interaction_range`、`block_break_speed`、`mining_efficiency`、`submerged_mining_speed`、`sneaking_speed`、`sweeping_damage_ratio`。
- 物品 / 职业可直接声明，如物品 `providers.ap.value` 写 `"scale":1.2`。旧版本服务器没有的属性自动跳过，零报错。

### 新增 — PAPI 占位符（冷却缩减 / 通用减免）

- `%qinhcorelib_cdr%`：冷却缩减值（0~0.9）。
- `%qinhcorelib_cdrmult%`：`1 - 冷却缩减` 乘子。
- `%qinhcorelib_cdrapply_<基础冷却>%`：**已应用缩减后的冷却值**。MythicMobs 技能可直接 `Cooldown: "%qinhcorelib_cdrapply_100%"` 吃到缩减——**不依赖 QinhSkills**，纯 MM + 核心库即可用。
- `%qinhcorelib_reduceapply_<属性>_<基础值>%`：通用「按某减免类属性削减基础值」，可复用于任意 0~1 减免属性。

### 修复 — 向上兼容门

- **启动版本校验改为「向上 fail-open」**：旧逻辑在解析不出服务端版本号（新格式 / 非标准发行版）时会判「无法识别」**直接禁用插件**，导致更高版本服务端误报不兼容、连带全家桶（依赖本库的 QI/QC/QS/QR）一起无法启动。现改为：解析失败 / 未知 / 更高版本一律**放行**，只拦截「确定低于 1.21.11」的过旧版本。
- 高于「已测试版本」时只**打印警告**并照常启动；想要严格行为可在 `config.yml` 开 `compatibility.strict-version-check: true`（默认关），开启后高于已测试版本才会拒绝。

### 其它

- 自定义属性 hook 列表新增 `on_magic_damage_dealt`（可在 `attributes.yml` 给自定义属性绑定法术伤害钩子）。

## [1.1.0] - 2026-06-14

### 新增

- **ItemsAdder 物品源**：引用格式 `ia-命名空间_id` 或 `itemsadder:命名空间:id`。
- **Nexo 物品源**：引用格式 `nexo-id` 或 `nexo:id`。
- 以上两源均为**反射桥**，对应插件未安装时自动跳过（降级，不报错）。
- `ItemSourceType` 枚举新增 `ITEMSADDER` / `NEXO` 两类。
- 所有走 `ItemSourceManager.parseItemReference` 的调用方**自动支持**这两个新源，子插件无需改动代码。
- `plugin.yml` 的 `softdepend` 增加 `ItemsAdder` / `Nexo`。
