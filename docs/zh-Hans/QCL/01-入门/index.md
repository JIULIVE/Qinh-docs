# 📖 入门

**QinhCoreLib（简称 QCL）是整个「秦淮生态」的核心底座 / 平台库。** 它本身不实现任何具体玩法，而是为上层玩法插件（QI 物品、QS 技能、QF 锻造、QSt 强化、QCR 等）以及第三方插件提供一整套统一能力：物品源、经济桥、脚本、数据库、PDC（持久化数据）、占位符、自定义 GUI、动作契约、桥接状态与诊断。

可以把它理解成「水电煤」——装了 QCL 不会多出任何玩法，但缺了它，秦淮系的玩法插件一个都跑不起来。所有秦淮系玩法插件都**硬依赖** QCL，第三方插件（Vault、PlaceholderAPI、ModelEngine、MMOItems 等）则通过它提供的反射桥接被「拉进来」统一使用，未安装时自动跳过。

因为是底座，QCL 应当**最先安装**：它本身没有任何硬依赖，但依赖 Paper / Purpur 的 `libraries` 机制来自动拉取 Kotlin、GraalJS、Groovy4 等运行库，所以服务端、Java、MC 版本都有明确要求。装好之后，先用 `/qcl status` 给整个生态做一次体检，再一步步看懂它的运作方式。

如果你是第一次接触，建议从「概览」读起，了解 QCL 在生态中的定位；接着按「安装与环境」搭好底座；用「快速上手」跑通第一遍；最后通过「核心概念与架构」理解模块、桥、物品源这几个贯穿全书的抽象。
