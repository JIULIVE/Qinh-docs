# Server Owner Guide Overview

> Previous: [Core Concepts & Architecture](../01-getting-started/core-concepts.md)　·　Next: [Ruin Template (full template.yml fields)](./ruin-template.md)

This is the **home page of the Server Owner Guide (zone 02)**. This zone teaches you how to take a ruin all the way from "a pile of files on disk" to "a complete experience where players stumble into it in the world, clear it, and grind tiers."

::: tip Tip
Still fuzzy on terms like "template / anchor / placement profile / blueprint / realm"? Read [Core Concepts & Architecture](../01-getting-started/core-concepts.md) first; coming back to this zone afterward will go a lot more smoothly.
:::

## 🗺️ A Ruin from Nothing

The flowchart below strings together all the chapters of the Server Owner Guide into a single pipeline. **Top to bottom is the actual order in which you configure a ruin**, with each step linking to its corresponding chapter.

```text
 ① Prepare the structure body (blocks)
    ├─ Select an existing build in the world ──→ /qr pos1 /qr pos2 /qr save
    └─ Import a WorldEdit .schem ──────────────→ /qr import
                        │
                        ▼
 ② Write the ruin template template.yml (its ID: what it looks like / which structure / where it generates / what it drops)
                        │
            ┌───────────┼─────────────┐
            ▼           ▼             ▼
 ③ Configure "where"  ④ Configure "gameplay"   ⑤ Configure "output"
   Placement Profile    Blueprint               Loot
   generators/*.yml     blueprint.yml           loottables/*.yml
            │           │             │
            └───────────┼─────────────┘
                        ▼
 ⑥ Preview & deploy: /qr stage ghost preview → confirm; or hand it to the Director for natural generation
                        │
                        ▼
 ⑦ Endgame: player clears → right-clicks the core with a Keystone → injects tiers + affixes = Realm
                        │
                        ▼
 ⑧ Player experience: guide compass points the way, codex collection, party-shared progress, bossbar
```

## 🚦 Where Should I Start?

- **Already have a finished build in hand** → first read [Selection & Saving](./selection-save.md) to store it as a template, then return to [Ruin Template](./ruin-template.md) to flesh out the fields.
- **Have someone else's `.schem` schematic** → read the `/qr import` section in [Structure Files](./structure-files.md).
- **Starting from scratch, want to understand the whole setup** → follow the flowchart above in order ①→⑧, starting from [Ruin Template](./ruin-template.md).
- **Just want to fatten up an existing ruin / add endgame** → jump straight to [Realms & Keystones](./realms-keystones.md) and [Loot System](./loot-tables.md).

## 🔧 Don't Forget After Configuring

- After editing any YAML (template / blueprint / profile / loot table / affix) → `/qr reload` to apply.
- To confirm whether a ruin can generate where you stand → `/qr why <template>` / `/qr gentest`, see [Diagnostics & Troubleshooting](../05-reference/troubleshooting.md).
- Quick reference for all commands, permissions, and config keys → [Command Reference](../05-reference/commands.md) · [config.yml full configuration](../05-reference/config.md).
