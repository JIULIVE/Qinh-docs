# Permission Nodes

> Belongs to: [Reference](./commands.md)　·　Related: [Commands](./commands.md)

From `plugin.yml` and the command code.

| Permission | Default | Sub-permissions | Description |
|---|---|---|---|
| `qinhitems.use` | OP | — | Basic commands (list, etc.) |
| `qinhitems.give` | OP | — | Give items |
| `qinhitems.reload` | OP | — | Reload config |
| `qinhitems.inspect` | OP | — | View info / diagnostics |
| `qinhitems.editor` | OP | — | Open the editor GUI |
| `qinhitems.admin` | OP | use / give / reload / inspect / editor | Full admin permission (parent node) |
| `qinhitems.bypass.soulbound` | — | — | Bypass all soulbinding restrictions (drop / pickup / breaking containers) |

> `qinhitems.admin` is a parent permission; granting it includes all sub-permissions: use / give / reload / inspect / editor.
>
> The exact node for `qinhitems.bypass.soulbound` can be changed via `soulbound.bypass-permission` in `config.yml`; see [Soulbinding](../02-server-guide/soulbinding.md#4-配置项).
