# 📚 Reference

This is the quick-lookup zone you keep coming back to while configuring QinhItems. The earlier chapters walk you from install through advanced play; this section is not a walkthrough but a lookup table: how to type a command, how to grant a permission, what every `config.yml` field means, what each placeholder is called, and which validation error maps to which fix, plus common questions and a glossary.

The pages are organized by the kind of question you have. Need a command? See [Commands](./commands) and [Permissions](./permissions). Editing config? Use [Config](./config) (grouped by topic) or [Annotated Config](./annotated-config) (line by line against the default file). Unsure what a given YAML file does? Check [Resource Files](./resource-files). When a red error shows up, look it up in [Validation Errors](./validation-errors) first, then reach for the commands in [Diagnostics](./diagnostics) if you still cannot pin it down.

Read these pages **on demand — you do not need to read them cover to cover**. Bookmark this index, then jump into the matching child page whenever a concrete question comes up.

- 📈 [Performance & Ops](./performance-ops) — Loading, refresh, and cache tuning for large item counts and high player counts.
- ❓ [FAQ](./faq) — Common pitfalls gathered from across the docs into grouped questions and answers.
- 📖 [Glossary](./glossary) — QI's domain terms (definition / layer / bridge / capability, etc.) explained in one line each, with links.
