# Glossary

> Belongs to: [Reference](./commands.md)　·　Back: [Home](../README.md)

Sorted by pinyin / alphabet — look anything up here whenever you hit an unfamiliar term.

| Term | Meaning | See |
|---|---|---|
| **AP / AttributePlus** | Third-party attribute plugin, QI's numbers backend | [Attributes & Numbers](../02-server-guide/attributes-numbers.md) |
| **Affix** | Reusable modifier pack (prefix / suffix / Lore) | [Affixes](../02-server-guide/affixes.md) |
| **Assembly** | The process of composing a template + instance + layers into an ItemStack | [Layers & Assembly](../04-developer/layers-assembly.md) |
| **base_values** | Core attributes managed by QI, compiled into ap | [Attributes & Numbers](../02-server-guide/attributes-numbers.md) |
| **Bridge** | Adapter that interprets a Provider payload | [Provider & Bridge](../04-developer/providers-bridges.md) |
| **Capability** | Functionality supported by a type (SKILL/ATTRIBUTE/GEM_SOCKET…) | [Item Types](../02-server-guide/item-types.md) |
| **CMD / CustomModelData** | Custom model data | [Resource Pack](../02-server-guide/resource-pack.md) |
| **Definition (template)** | The item blueprint after a YAML is compiled | [Item Definition](../02-server-guide/item-definition.md) |
| **Fragment** | Reusable item YAML snippet | [Fragments & Templates](../02-server-guide/fragments-templates.md) |
| **Handler** | The effect unit executed after a trigger fires | [Handlers](../02-server-guide/action-system/handlers.md) |
| **ICVM** | QI's internal standard vocabulary for attributes | [Core Concepts](../01-getting-started/core-concepts.md) |
| **Instance (instance data)** | Mutable state written into a single ItemStack's NBT | [Core Concepts](../01-getting-started/core-concepts.md) |
| **Layer** | A patch appended after creation (socketing / enhancement, etc.) | [Layers & Assembly](../04-developer/layers-assembly.md) |
| **LibraryManifest** | Content pack metadata | [Fragments & Templates](../02-server-guide/fragments-templates.md) |
| **Provider** | An external system payload attached to an item | [Provider & Bridge](../04-developer/providers-bridges.md) |
| **QCL / QinhCoreLib** | Qinhuai's unified low-level library, a hard dependency of QI | [Overview](../01-getting-started/overview.md) |
| **QS / QinhSkills** | Skill engine, hooked via `qinhskills:cast` | [Integration](../04-developer/integration.md) |
| **Section** | Reusable Lore block / random pool | [Sections](../02-server-guide/sections.md) |
| **Set** | Bonuses activated by the number of pieces worn | [Sets](../02-server-guide/sets.md) |
| **Soulbound** | An item bound to a player and untradeable | [Soulbinding](../02-server-guide/soulbinding.md) |
| **Tier (quality)** | Rarity level | [Quality & Display](../02-server-guide/quality-display.md) |
| **Trigger / Atom** | The player behavior that fires an action | [Triggers](../02-server-guide/action-system/triggers.md) |
| **Variable** | Item-level dynamic value, rendered via the `{变量}` placeholder | [Variables](../02-server-guide/variables.md) |
| **WriteDomain** | Policy restricting who can change which part of an item | [Layers & Assembly](../04-developer/layers-assembly.md) |
| **.qipack** | QI content pack export format | [Import & Export](../04-developer/import-export.md) |
| **Pure item-library mode** | Run mode that only crafts items without adding attributes when AP is absent | [Attributes & Numbers](../02-server-guide/attributes-numbers.md) |
