# Editable Messages / Text Reference

> Section: [Reference](./commands.md)гҖҖВ·гҖҖRelated: [Config File](./config.md) В· [Soulbound](../02-server-guide/soulbinding.md)

This page gathers all the **server-customizable prompt text / display formats** in QI, making localization / tone adjustment easy. Most support `&` color codes or MiniMessage, plus placeholders.

---

## 1. Soulbound Text (config.yml вҶ’ soulbound)

| Key | Default | Placeholders |
|---|---|---|
| `lore-format` | `&7{owner}'s item` | `{owner}` `{level}` |
| `messages.drop` | `&cThis item is soulbound and cannot be dropped.` | вҖ” |
| `messages.pickup` | `&cThis item belongs to {owner}, you cannot pick it up.` | `{owner}` |
| `messages.move` | `&cThis item belongs to {owner}, you cannot move it.` | `{owner}` |
| `messages.container` | `&cThis container holds bound items and cannot be broken.` | вҖ” |
| `messages.container-drop` | `&c"{container}" holds bound items and cannot be dropped.` | `{container}` |

Leave `lore-format` empty to hide the binding line. See [Soulbound](../02-server-guide/soulbinding.md).

## 2. Set Lore Format (sets/*.yml вҶ’ lore_format)

Customizable per set, with variables `{name}` `{pieces}` `{need}` `{lore}` `{desc}`:

| Key | Default |
|---|---|
| `header` | `{name}` |
| `inactive_line` | `в—Ү [{pieces}] {name} [{need} more needed]` |
| `active_line` | `в—Ҷ [{pieces}] {name} [Activated]` |
| `lore_line` | `  {lore}` |
| `lore_divider` | `в”Җ Set Description в”Җ` |
| `desc_line` | `  {desc}` |

See [Sets вҶ’ Lore Display](../02-server-guide/sets.md#6-еҘ—иЈ…-lore-жҳҫзӨә).

## 3. Tier Display (item_tiers.yml)

Each Tier's `name` (colored display name), `unidentification.name` (unidentified name), and `unidentification.prefix` are all editable. See [Tiers & Display](../02-server-guide/quality-display.md).

---

## 4. Attribute Display Labels

Attribute lore labels come from the built-in `ApDisplayLabels` (e.g. `attack_damage` вҶ’ Attack Damage). **These are hardcoded in the code** and are not changed in YAML; but you can:

- Use `attribute-mapping` to change the ICVM вҶ’ AP display name mapping (affects the name handed off to AP).
- Write whatever text you want directly in the item lore (`lore:` field).

See [Tiers & Display вҶ’ Attribute Display Labels](../02-server-guide/quality-display.md#6-еұһжҖ§жҳҫзӨәж Үзӯҫ) and [Attributes & Values](../02-server-guide/attributes-numbers.md).

---

## 5. Action Runtime Prompts (config.yml вҶ’ actions)

| Key | Default | Effect |
|---|---|---|
| `notify-cooldown` | `true` | Whether to prompt the player when triggered during cooldown |
| `notify-handler-unavailable` | `true` | Whether to prompt when the handler is unavailable |

The prompt text itself is built into `ActionPlayerFeedback`. To fully customize these prompts, turn off the switch and output via your own action handler.

---

## 6. Item Text (written per item)

All item-level visible text lives in the item YAML; change it freely:

- `display_name` вҖ”вҖ” item name
- `lore` вҖ”вҖ” description lines
- Action `qi:message` / `qi:title` / `qi:action_bar` etc. payloads вҖ”вҖ” skill prompt text
- Section / affix `prefix` / `suffix` / `lore` вҖ”вҖ” prefix/suffix and affix descriptions

All support MiniMessage (`<gold>`, `<gradient>`, etc.) and legacy `&` color codes.

---

## 7. Language

`config.yml`'s `language: zh_CN` sets the message language code.

---

## Localization / Text Editing Checklist

1. `config.yml` вҶ’ `soulbound.messages.*` / `lore-format`
2. `sets/*.yml` вҶ’ each set's `lore_format`
3. `item_tiers.yml` вҶ’ each Tier's `name`
4. `items/*.yml` вҶ’ each item's `display_name` / `lore` / action payload
5. `sections/*.yml` вҶ’ section / affix text
6. `config.yml` вҶ’ `actions.notify-*` switches
