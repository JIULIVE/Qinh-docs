# 📚 Reference

The Reference section is QCL's quick-lookup zone. When you don't want to read a tutorial from the top and just need to confirm a term, an error code, or what a given release changed, this is where you come. It gathers the glossary that runs through the whole Qinh RPG ecosystem, the topic-grouped FAQ, the master table of diagnostic and error codes across subsystems, and a changelog that follows semantic versioning.

How to use it is straightforward: when you hit an unfamiliar abbreviation (QI, QS, QF, and the like), check the glossary first; for questions around installation, item sources, economy, or scripts, browse the FAQ; when `/qcl status` shows red or an API returns a non-OK code, look the code up in the diagnostics table for its meaning and suggested fix; and when you want to know which release introduced a reference format or behavior, read the changelog.

## In this section

- 📖 [Glossary](./glossary) — one-line explanations of common QCL and Qinh ecosystem terms, each pointing to a detail page.
- ❓ [FAQ](./faq) — frequent questions grouped by installation, item sources, economy, GUI/scripts, operations, and developers.
- 🚨 [Diagnostic Codes](./diagnostic-codes) — master table of subsystem status and error codes, with triggers and suggested fixes.
- 📅 [Changelog](./changelog) — version history following semantic versioning (SemVer).
